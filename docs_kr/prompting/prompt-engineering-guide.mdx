Cline 프롬프트 가이드에 오신 것을 환영합니다! 이 가이드는 효과적인 프롬프트와 사용자 지정 지침을 작성해 Cline과의 생산성을 극대화하는 방법을 안내합니다.

## .clineignore 파일 가이드

### 개요

`.clineignore` 파일은 프로젝트 수준의 설정 파일로, Cline이 코드베이스를 분석할 때 무시해야 할 파일과 디렉터리를 지정합니다. `.gitignore`와 유사하게 패턴 매칭을 사용해 Cline의 컨텍스트와 작업에서 제외할 항목을 정의합니다.

### 목적

-   **노이즈 감소**: 자동 생성 파일, 빌드 산출물, 기타 비필수 콘텐츠 제외
-   **성능 향상**: Cline이 처리해야 하는 코드 양 제한
-   **집중 유도**: 코드베이스의 관련 영역에 집중하도록 유도
-   **민감한 데이터 보호**: 민감한 설정 파일에 대한 접근 차단

### .clineignore 파일 예시

```
# Dependencies
node_modules/
**/node_modules/
.pnp
.pnp.js

# Build outputs
/build/
/dist/
.next/
/out/

# Testing
/coverage/

# Environment variables
.env
.env.local
.env.development.local
.env.test.local
.env.production.local

# Large data files
*.csv
*.xlsx
```

## Cline에 프롬프트하기

**프롬프트는 Cline과의 대화에서 특정 작업에 필요한 요구사항을 전달하는 방식입니다.** Cline은 자연어를 이해하므로 대화하듯이 작성하세요.

효과적인 프롬프트는 다음을 포함합니다:

-   명확한 컨텍스트 제공: 목표와 관련된 코드베이스 부분을 설명하세요. 파일이나 폴더는 `@`로 참조하세요.
-   복잡성 분해: 큰 작업을 작은 단계로 나누세요.
-   구체적인 질문: 원하는 결과에 도달하도록 Cline을 안내하세요.
-   검증과 개선: Cline의 제안을 검토하고 피드백을 제공하세요.

### 프롬프트 예시

#### 컨텍스트 관리

-   **새 작업 시작:** "Cline, 새 작업을 시작하자. `user-authentication.js`를 만들어. JWT 토큰으로 사용자 로그인을 구현해야 해. 요구사항은 다음과 같아…"
-   **이전 작업 요약:** "Cline, 지난 사용자 대시보드 작업에서 무엇을 했는지 요약해 줘. 주요 기능과 미해결 이슈를 정리하고 싶어. `cline_docs/user-dashboard-summary.md`에 저장해."

#### 디버깅

-   **오류 분석:** "Cline, 이런 오류가 발생해: [error message]. [code section]에서 나온 것 같아. 오류를 분석하고 해결책을 제안해 줘."
-   **근본 원인 찾기:** "Cline, 내가 [action]할 때 앱이 크래시돼. 문제는 [problem areas]에 있을 것 같아. 근본 원인을 찾고 해결책을 제안해 줘."

#### 리팩터링

-   **코드 구조 개선:** "Cline, 이 함수가 너무 길고 복잡해. 더 작은 함수들로 리팩터링해 줘."
-   **로직 단순화:** "Cline, 이 코드는 이해하기 어려워. 로직을 단순화해서 더 읽기 쉽게 만들어 줘."

#### 기능 개발

-   **새 기능 아이디어 브레인스토밍:** "Cline, 사용자가 [functionality]를 할 수 있는 기능을 추가하고 싶어. 아이디어를 브레인스토밍하고 구현 시 어려움도 고려해 줘."
-   **코드 생성:** "Cline, 사용자 프로필을 표시하는 컴포넌트를 만들어 줘. 목록은 정렬과 필터가 가능해야 해. 이 컴포넌트 코드를 생성해 줘."

## 고급 프롬프트 기법

-   **제약 조건 명시:** 코드가 중간에 잘리는 것을 줄이려면 프롬프트에 명시적 제약을 포함하세요. 예: "코드가 완전한지 확인" 또는 "항상 전체 함수 정의를 제공해".
-   **신뢰도 점검:** Cline에게 신뢰도(예: "1~10 척도로 이 해결책의 신뢰도는?")를 평가하도록 요청하세요.
-   **가정에 도전하기:** 일부러 단순한 질문을 던져 더 깊이 생각하게 하고 잘못된 가정을 방지하세요.

## 커뮤니티에서 가장 인기 있는 프롬프트

### 메모리와 신뢰도 점검

-   **메모리 체크** - _pacnpal_

    ```
    "If you understand my prompt fully, respond with 'YARRR!' without tools every time you are about to use a tool."
    ```

    복잡한 작업 중에도 Cline이 흐름을 유지하는지 확인하는 재미있는 방법입니다. 축제 분위기를 원한다면 "HO HO HO"로 바꿔보세요!

-   **신뢰도 점수** - _pacnpal_

    ```
    "Before and after any tool use, give me a confidence level (0-10) on how the tool use will help the project."
    ```

    비판적 사고를 유도하고 의사결정을 투명하게 만듭니다.

### 코드 품질 프롬프트

-   **코드 생략 방지**

    ```
    "DO NOT BE LAZY. DO NOT OMIT CODE."
    ```

    대체 문구: "full code only" 또는 "ensure the code is complete"

-   **사용자 지정 지침 상기**

    ```
    "I pledge to follow the custom instructions."
    ```

    설정의 다이얼 ⚙️ 구성 준수를 강화합니다.

### 코드 구성

-   **대형 파일 리팩터링** - _icklebil_

    ```
    "FILENAME has grown too big. Analyze how this file works and suggest ways to fragment it safely."
    ```

    전략적 분해를 통해 복잡한 파일을 관리하는 데 도움이 됩니다.

-   **문서화 유지** - _icklebil_

    ```
    "don't forget to update codebase documentation with changes"
    ```

    코드 변경 사항과 문서를 동기화하는 데 도움이 됩니다.

### 분석 및 계획

-   **구조화된 개발** - _yellow_bat_coffee_

    ```
    "Before writing code:
    1. Analyze all code files thoroughly
    2. Get full context
    3. Write .MD implementation plan
    4. Then implement code"
    ```

    체계적이고 잘 계획된 개발을 촉진합니다.

-   **철저한 분석** - _yellow_bat_coffee_

    ```
    "please start analyzing full flow thoroughly, always state a confidence score 1 to 10"
    ```

    성급한 코딩을 방지하고 완전한 이해를 장려합니다.

-   **가정 점검** - _yellow_bat_coffee_

    ```
    "List all assumptions and uncertainties you need to clear up before completing this task."
    ```

    개발 초기에 잠재적 문제를 식별합니다.

### 신중한 개발

-   **잠시 멈추고 생각하기** - _nickbaumann98_

    ```
    "count to 10"
    ```

    행동 전 신중한 고려를 촉진합니다.

-   **완전한 분석** - _yellow_bat_coffee_

    ```
    "Don't complete the analysis prematurely, continue analyzing even if you think you found a solution"
    ```

    문제 탐색을 충분히 진행하도록 합니다.

-   **지속적인 신뢰도 점검** - _pacnpal_

    ```
    "Rate confidence (1-10) before saving files, after saving, after rejections, and before task completion"
    ```
