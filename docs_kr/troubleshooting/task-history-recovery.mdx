Cline을 업데이트하거나 특정 설정이 변경되면 이전 작업에 접근하지 못하는 경우가 있습니다. 이 가이드는 Cline 작업 기록을 복구하고 재구성해 중요한 대화와 작업을 다시 확인할 수 있도록 도와줍니다.

<Tip>
	대부분의 경우 내장 복구 명령을 실행하면 해결됩니다.
</Tip>

## 빠른 복구

<Steps>
	<Step title="명령 팔레트 열기">
		`Cmd/Ctrl + Shift + P`를 눌러 Command Palette를 엽니다.
	</Step>
	<Step title="복구 명령 실행">
		**"Cline: Reconstruct Task History"**를 입력하고 선택합니다.
	</Step>
	<Step title="작업 확인">
		확인 프롬프트가 표시됩니다. **Yes**를 클릭합니다.
	</Step>
	<Step title="재구성 대기">
		Cline이 작업 폴더를 스캔해 기록 인덱스를 재구성합니다.
	</Step>
	<Step title="복구 확인">
		히스토리 패널에서 작업이 복구되었는지 확인합니다.
	</Step>
</Steps>

## Cline의 데이터 저장 위치

### 저장 경로

<Tabs>
	<Tab title="VS Code">
		```bash
		# macOS
		~/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/
		# Windows
		%APPDATA%\Code\User\globalStorage\saoudrizwan.claude-dev\
		# Linux
		~/.config/Code/User/globalStorage/saoudrizwan.claude-dev/
		```
	</Tab>
	<Tab title="JetBrains">
		```bash
		# macOS
		~/Library/Application Support/JetBrains/<IDE>/globalStorage/saoudrizwan.claude-dev/
		# Windows
		%APPDATA%\JetBrains\<IDE>\globalStorage\saoudrizwan.claude-dev\
		# Linux
		~/.config/JetBrains/<IDE>/globalStorage/saoudrizwan.claude-dev/
		```
	</Tab>
</Tabs>

VS Code Insiders를 사용 중이면 `Code`를 `Code - Insiders`로 바꾸세요. JetBrains IDE는 `<IDE>`를 사용 중인 IDE 이름으로 바꾸세요(예: `IntelliJIdea2023.3`, `PyCharm2023.3`, `WebStorm2023.3`).

### 디렉터리 구조

```
saoudrizwan.claude-dev/
├── state/
│   ├── taskHistory.json          # 메인 기록 인덱스
│   └── taskHistory.backup.*.json # 백업
├── tasks/
│   └── <task-id>/                # 개별 작업 데이터
│       ├── api_conversation_history.json
│       ├── ui_messages.json
│       └── task_metadata.json
└── checkpoints/
    └── <workspace-hash>/         # 워크스페이스별 체크포인트 저장소
        └── .git/                 # 스냅샷용 섀도 Git 저장소
```

`taskHistory.json` 파일은 인덱스일 뿐입니다. 실제 대화 데이터는 `tasks/` 아래의 개별 작업 폴더에 있습니다.

## 복구 명령 사용하기

복구 명령은 모든 작업 폴더를 스캔해 기록 인덱스를 처음부터 다시 만듭니다.

동작 방식:

1. 현재 `taskHistory.json` 백업
2. `tasks/` 디렉터리 스캔
3. 각 작업 폴더의 대화 데이터 읽기
4. 토큰 사용량과 비용 재계산
5. 새 `taskHistory.json` 생성

```mermaid
graph LR
    A[현재 인덱스 백업] --> B[tasks/ 스캔]
    B --> C[각 작업 읽기]
    C --> D[인덱스 재구성]
    D --> E[완료]
```

<Warning>
	복구 명령은 기존 작업 폴더의 데이터를 읽어 인덱스를 재구성합니다. `tasks/` 디렉터리 또는 특정 작업 폴더가 영구 삭제된 경우 해당 데이터는 복구할 수 없습니다.
</Warning>

## 수동 복구

### 백업에서 복원

Cline은 자동으로 백업을 생성합니다. `state/` 폴더에서 찾을 수 있습니다:

<Tabs>
	<Tab title="VS Code">
		```bash
		# macOS/Linux
		cd ~/.config/Code/User/globalStorage/saoudrizwan.claude-dev/state/
		ls taskHistory.backup.*.json

		# 가장 최신 파일을 선택해 복원
		cp taskHistory.backup.1234567890.json taskHistory.json
		```
	</Tab>
	<Tab title="JetBrains">
		```bash
		# macOS
		cd ~/Library/Application\ Support/JetBrains/<IDE>/globalStorage/saoudrizwan.claude-dev/state/
		ls taskHistory.backup.*.json

		# 가장 최신 파일을 선택해 복원
		cp taskHistory.backup.1234567890.json taskHistory.json
		```
	</Tab>
</Tabs>

### 새 머신으로 이전

새 컴퓨터로 옮기시나요? Cline 대화를 모두 가져올 수 있습니다. VS Code나 JetBrains IDE 모두 동일한 절차입니다.

<Steps>
	<Step title="Cline 데이터 찾기 및 복사">
		**기존 머신**에서 [위 저장 경로](#storage-paths)를 참고해 `saoudrizwan.claude-dev` 폴더를 찾고 전체를 복사합니다.
	</Step>
	<Step title="새 머신 준비">
		**새 머신**에 IDE(VS Code 또는 JetBrains)를 설치하고 Cline 확장을 설치합니다.
	</Step>
	<Step title="IDE 종료">
		진행하기 전에 IDE를 완전히 종료하세요.
	</Step>
	<Step title="데이터 이동">
		복사한 `saoudrizwan.claude-dev` 폴더를 새 머신의 동일한 저장 경로에 붙여넣습니다.
	</Step>
	<Step title="실행 및 확인">
		IDE를 열고 Cline의 작업 기록이 나타나는지 확인합니다.
	</Step>
</Steps>

<Info>
	데이터 형식은 운영체제 간에 동일합니다. 크로스 플랫폼 마이그레이션(예: Windows → macOS)도 추가 단계 없이 동작합니다.
</Info>

## 일반적인 문제

다음은 흔한 문제와 해결책입니다:

### VS Code 업데이트 후 기록이 비어 있음
Command Palette에서 **"Cline: Reconstruct Task History"**를 실행하세요. 그래도 안 되면 백업 파일을 복원할 수 있는지 확인하세요.

### VS Code 재설치 후 기록 분실
VS Code는 보통 확장 데이터를 유지합니다. Cline을 다시 설치한 다음 복구 명령을 실행하세요.

### "No tasks found" 오류
`tasks/` 폴더가 비어 있거나 누락된 경우 발생합니다. 주요 원인:

- VS Code 데이터가 완전히 삭제됨
- 다른 VS Code 설치(일반 버전 vs Insiders)를 보고 있음
- 폴더가 수동 삭제됨

IDE에 맞는 저장 경로를 확인하고 폴더가 존재하는지 확인하세요.

### 복구는 되지만 일부 작업이 누락됨
손상된 작업 폴더는 복구 중 건너뛰어집니다. 복구된 작업 수와 스킵된 작업 수가 표시됩니다. 어떤 작업이 복원되지 않았는지 오류 메시지를 확인하세요.
