## 먼저 시도할 것: 백그라운드 실행 모드

터미널 문제의 가장 간단한 해결책은 **Background Execution Mode**로 전환하는 것입니다:

1. **Settings** 클릭(Cline 채팅 오른쪽 상단)
2. **Terminal Settings**로 이동
3. **Terminal Execution Mode** → **Background Exec**로 설정

이 모드는 VSCode 통합 터미널 대신 백그라운드 프로세스에서 명령을 실행해 대부분의 통합 문제를 우회합니다.

---

## 다른 해결책

VSCode 통합 터미널이 필요하다면 다음을 시도하세요:

-   **bash로 전환** (대부분의 경우 해결됨)

    1. Cline 채팅 오른쪽 상단의 **Settings** 버튼 클릭
    2. **Settings** 창 왼쪽의 **"Terminal Settings"** 탭 클릭
    3. **"Default Terminal Profile"**에서 **"bash"** 선택

-   **타임아웃 증가**

    1. Cline 채팅 오른쪽 상단의 **Settings** 버튼 클릭
    2. **Settings** 창 왼쪽의 **"Terminal Settings"** 탭 클릭
    3. "Shell integration timeout (seconds)"에 **"10"** 입력

-   **터미널 재사용 비활성화**
    1. Cline 채팅 오른쪽 상단의 **Settings** 버튼 클릭
    2. **Settings** 창 왼쪽의 **"Terminal Settings"** 탭 클릭
    3. **"Enable aggressive terminal reuse"** 옵션을 **해제**

## 플랫폼별 해결책

### macOS + Oh-My-Zsh

```bash
# VSCode용 최소 설정 생성
echo 'export TERM=xterm-256color' > ~/.zshrc-vscode
echo 'export PAGER=cat' >> ~/.zshrc-vscode
```

### Windows PowerShell

```powershell
# 관리자 권한으로 실행
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### WSL

-   WSL에서 폴더 열기: `code .`
-   Cline 설정의 **"Terminal Settings"**에서 **"WSL Bash"** 선택
-   **"Shell integration timeout (seconds)"**를 **15**로 증가
