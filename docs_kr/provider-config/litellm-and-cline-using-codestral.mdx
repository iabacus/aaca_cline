### Cline에서 LiteLLM 사용하기

이 가이드는 Codestral 모델을 사용해 Cline과 함께 사용할 LiteLLM 데모를 실행하는 방법을 보여줍니다.

#### 사전 준비

-   로컬에서 LiteLLM 이미지를 실행하기 위한 [Docker CLI 또는 Docker Desktop](https://www.docker.com/get-started/) 설치
-   이 예시 구성에 필요한 Codestral API 키(Mistral API 키와는 다름)

#### 설정

1. **`.env` 파일을 만들고 필요한 값을 입력합니다**

    ```bash
    # 팁: 다음 명령으로 임의의 영숫자 키를 생성할 수 있습니다:
    # openssl rand -base64 32 | tr -dc 'A-Za-z0-9' | head -c 32
    LITELLM_MASTER_KEY=YOUR_LITELLM_MASTER_KEY
    CODESTRAL_API_KEY=YOUR_CODESTRAL_API_KEY
    ```

    _참고: localhost에만 제한하더라도 LITELLM_MASTER_KEY는 안전한 값으로 설정하는 것이 좋습니다._

2. **구성**

    LiteLLM 구성을 담을 `config.yaml` 파일을 만듭니다. 이 경우 모델은 하나만 사용하며 `codestral-latest`를 `codestral`로 라벨링합니다.

    ```yaml
    model_list:
        - model_name: codestral
          litellm_params:
              model: codestral/codestral-latest
              api_key: os.environ/CODESTRAL_API_KEY
    ```

#### 데모 실행

1. **LiteLLM Docker 컨테이너 시작**

    ```bash
    docker run \
        --env-file .env \
        -v $(pwd)/config.yaml:/app/config.yaml \
        -p 127.0.0.1:4000:4000 \
        ghcr.io/berriai/litellm:main-latest \
        --config /app/config.yaml --detailed_debug
    ```

2. **Cline 설정**

    LiteLLM 서버가 실행되면 Cline에서 다음과 같이 설정합니다:

    - Base URL은 `http://0.0.0.0:4000/v1`
    - API Key는 `.env`에서 설정한 LITELLM_MASTER_KEY
    - Model ID는 `codestral` 또는 `config.yaml`에서 지정한 이름
