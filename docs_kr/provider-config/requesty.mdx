Cline은 [Requesty](https://www.requesty.ai/) AI 플랫폼을 통해 모델에 접근할 수 있습니다. Requesty는 150개 이상의 대형 언어 모델(LLM)과 상호작용할 수 있는 간단하고 최적화된 API를 제공합니다.

**웹사이트:** [https://www.requesty.ai/](https://www.requesty.ai/)

### API 키 받기

1.  **가입/로그인:** [Requesty 웹사이트](https://www.requesty.ai/)에 접속해 계정을 만들거나 로그인합니다.
2.  **API 키 받기:** Requesty 대시보드의 [API Management](https://app.requesty.ai/api-keys) 섹션에서 API 키를 받을 수 있습니다.

### 지원 모델

Requesty는 다양한 모델에 접근할 수 있습니다. Cline은 최신 모델 목록을 자동으로 가져옵니다. 전체 모델 목록은 [Model List](https://app.requesty.ai/router/list) 페이지에서 확인할 수 있습니다.

### Cline에서 설정하기

1.  **Cline 설정 열기:** Cline 패널의 설정 아이콘(⚙️)을 클릭합니다.
2.  **프로바이더 선택:** "API Provider" 드롭다운에서 "Requesty"를 선택합니다.
3.  **API 키 입력:** Requesty API 키를 "Requesty API Key" 필드에 붙여넣습니다.
4.  **모델 선택:** "Model" 드롭다운에서 원하는 모델을 선택합니다.

### 팁과 참고사항

-   **최적화**: Requesty는 다양한 인-플라이트 비용 최적화 기능으로 비용을 낮출 수 있습니다.
-   **통합/간소화된 결제**: 모든 프로바이더와 모델에 무제한 접근, 자동 잔액 충전 등 기능을 단일 [API 키](https://app.requesty.ai/api-keys)로 제공합니다.
-   **비용 추적**: [Cost dashboard](https://app.requesty.ai/cost-management) 또는 [Requesty VS Code 확장](https://marketplace.visualstudio.com/items?itemName=Requesty.requesty)을 통해 모델별, 언어별, 변경된 파일별 비용을 추적할 수 있습니다.
-   **통계 및 로그**: [코딩 통계 대시보드](https://app.requesty.ai/usage-stats) 또는 [LLM 상호작용 로그](https://app.requesty.ai/logs)를 확인하세요.
-   **폴백 정책**: 프로바이더 장애 시 폴백 정책으로 LLM 작업을 계속 유지할 수 있습니다.
-   **프롬프트 캐싱:** 일부 프로바이더는 프롬프트 캐싱을 지원합니다. [캐싱 지원 모델 검색](https://app.requesty.ai/router/list).
