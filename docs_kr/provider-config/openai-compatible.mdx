Cline은 OpenAI API 표준과 호환되는 API를 제공하는 다양한 AI 모델 프로바이더를 지원합니다. 이를 통해 OpenAI가 아닌 프로바이더의 모델을 사용하면서도 익숙한 API 인터페이스를 활용할 수 있습니다. 예시는 다음과 같습니다:

-   **로컬 모델**: Ollama, LM Studio 같은 도구를 통해 실행되는 모델(각 섹션 참고)
-   **클라우드 프로바이더**: Perplexity, Together AI, Anyscale 등 다양한 서비스
-   **기타 프로바이더**: OpenAI 호환 API 엔드포인트를 제공하는 모든 서비스

이 문서는 공식 OpenAI API가 아닌 프로바이더 설정에 집중합니다(공식 OpenAI API는 별도의 [전용 설정 페이지](/provider-config/openai)를 참고).

### 일반 설정

OpenAI 호환 프로바이더를 Cline에서 사용하려면 다음 핵심 설정을 구성해야 합니다:

1.  **Base URL:** 프로바이더의 API 엔드포인트입니다. `https://api.openai.com/v1`이 _아닙니다_ (해당 URL은 공식 OpenAI API용).
2.  **API Key:** 선택한 프로바이더에서 발급한 비밀 키입니다(또는 **Use Azure Identity Authentication** 사용).
3.  **Model ID:** 사용하려는 모델의 이름 또는 식별자입니다.

이 설정은 Cline 설정 패널(⚙️ 아이콘 클릭)에서 찾을 수 있습니다:

-   **API Provider:** "OpenAI Compatible" 선택
-   **Base URL:** 선택한 프로바이더가 제공한 base URL 입력(중요)
-   **API Key:** 프로바이더 API 키 입력
-   **Model:** 모델 ID 선택 또는 입력
-   **Use Azure Identity Authentication:** Azure 관리형 ID로 인증하려면 체크(이 옵션은 인증을 직접 트리거하지 않고 기존 인증 상태(예: `az login`)를 사용합니다)
-   **Model Configuration:** 다음과 같은 고급 파라미터를 설정할 수 있습니다:
    -   Max Output Tokens
    -   Context Window size
    -   Image Support capabilities
    -   Computer Use(도구/함수 호출 지원 모델 등)
    -   Input Price(토큰/백만 토큰당)
    -   Output Price(토큰/백만 토큰당)

### 지원 모델(OpenAI Native Endpoint용)

"OpenAI Compatible" 프로바이더는 다양한 엔드포인트에 연결할 수 있지만, 공식 OpenAI API(또는 이를 정확히 미러링한 엔드포인트)에 연결하는 경우 Cline은 소스 코드의 `openAiNativeModels` 정의를 기준으로 다음 모델 ID를 인식합니다:

-   `o3-mini`
-   `o3-mini-high`
-   `o3-mini-low`
-   `o1`
-   `o1-preview`
-   `o1-mini`
-   `gpt-4o`
-   `gpt-4o-mini`

**참고:** Together AI, Anyscale 등의 다른 OpenAI 호환 프로바이더를 사용하는 경우 모델 ID는 달라집니다. 지원 모델 이름과 구성 세부사항은 해당 프로바이더 문서를 참고하세요.

### v0 (Vercel SDK) in Cline:

-   v0를 사용하는 개발자라면 [AI SDK 문서](https://vercel.com/docs/v0/cline)에 다양한 모델 통합 예시가 있습니다. OpenAI 호환 모델이 많아 Cline과 Vercel 서비스 연동 시 유용합니다.

-   v0는 Cline에서 OpenAI Compatible 프로바이더로 사용할 수 있습니다.

-   ### 빠른 시작

-   1. OpenAI Compatible 프로바이더를 선택한 상태에서 Base URL을 https://api.v0.dev/v1 로 설정합니다.
-   2. v0 API 키를 붙여넣습니다.
-   3. Model ID를 v0-1.0-md 로 설정합니다.
-   4. Verify를 클릭해 연결을 확인합니다.

### 문제 해결

-   **"Invalid API Key":** API 키가 정확하고 올바른 프로바이더의 키인지 확인하세요.
-   **"Model Not Found":** 사용 중인 프로바이더에서 유효한 모델 ID인지, 지정한 Base URL에서 사용 가능한지 확인하세요.
-   **연결 오류:** Base URL이 올바른지, 프로바이더 API가 로컬에서 접근 가능한지, 방화벽/네트워크 문제가 없는지 확인하세요.
-   **예상치 못한 결과:** 다른 모델을 시도하거나 모든 구성 파라미터를 다시 확인하세요.

OpenAI 호환 프로바이더를 사용하면 Cline에서 더 넓은 AI 모델 선택지를 활용할 수 있습니다. 최신 정보는 각 프로바이더 문서를 참고하세요.
