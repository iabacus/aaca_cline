Z AI(구 Zhipu AI)는 하이브리드 추론 기능과 에이전트형 AI 설계를 갖춘 GLM-4.5 시리즈를 제공합니다. 2025년 7월 출시된 이 모델들은 통합 추론, 코딩, 지능형 에이전트 애플리케이션에서 뛰어난 성능을 보이며, MIT 라이선스 기반 오픈소스 접근성을 유지합니다.

**웹사이트:** [https://z.ai/model-api](https://z.ai/model-api) (International) | [https://open.bigmodel.cn/](https://open.bigmodel.cn/) (China)

### 지원 모델

Z AI는 선택한 지역에 따라 서로 다른 모델 카탈로그를 제공합니다:

#### GLM-4.5 시리즈
-   **GLM-4.5** - 총 355B 파라미터, 활성 32B 파라미터의 플래그십 모델
-   **GLM-4.5-Air** - 총 106B 파라미터, 활성 12B 파라미터의 컴팩트 모델

#### GLM-4.5 하이브리드 추론 모델
-   **GLM-4.5 (Thinking Mode)** - 단계별 분석을 포함한 고급 추론
-   **GLM-4.5-Air (Thinking Mode)** - 일반 하드웨어에 최적화된 효율적 추론

모든 모델 특징:
-   **128,000 토큰 컨텍스트 창**으로 대규모 문서 처리
-   **Mixture of Experts(MoE) 아키텍처**로 최적 성능
-   **에이전트 네이티브 설계**로 추론, 코딩, 도구 사용 통합
-   **MIT 라이선스 오픈소스** 제공

### Cline에서 설정하기

1.  **Cline 설정 열기:** Cline 패널의 설정 아이콘(⚙️)을 클릭합니다.
2.  **프로바이더 선택:** "API Provider" 드롭다운에서 "Z AI"를 선택합니다.
3.  **지역 선택:**
    -   글로벌 접근은 "International"
    -   중국 본토 접근은 "China"
4.  **API 키 입력:** Z AI API 키를 "Z AI API Key" 필드에 붙여넣습니다.
5.  **모델 선택:** "Model" 드롭다운에서 원하는 모델을 선택합니다.

### Z AI의 하이브리드 지능

Z AI의 GLM-4.5 시리즈는 기존 언어 모델과 차별화되는 혁신적인 기능을 제공합니다:

#### 하이브리드 추론 아키텍처
GLM-4.5는 두 가지 모드로 동작합니다:
- **Thinking Mode:** 복잡한 추론 및 도구 사용을 위한 깊은 분석
- **Non-Thinking Mode:** 간단한 질문에 즉시 응답하여 효율성 최적화

이 이중 모드 아키텍처는 쿼리 복잡도에 따라 처리 강도를 조정하는 "에이전트 네이티브" 설계 철학을 반영합니다.

#### 뛰어난 성능
GLM-4.5는 에이전트 작업, 추론, 코딩 과제 등 12개 벤치마크에서 **63.2** 종합 점수를 기록하며, 상용 및 오픈소스 모델 전체에서 **3위**를 차지했습니다. GLM-4.5-Air는 **59.8** 점으로 경쟁력 있는 성능을 유지하면서 효율성을 제공합니다.

#### MoE의 탁월함
정교한 MoE 아키텍처는 계산 효율을 유지하면서 성능을 최적화합니다:
- **GLM-4.5:** 총 355B 파라미터, 활성 32B 파라미터
- **GLM-4.5-Air:** 총 106B 파라미터, 활성 12B 파라미터

#### 확장된 컨텍스트
128,000 토큰 컨텍스트 창으로 긴 문서와 코드베이스를 포괄적으로 이해할 수 있으며, 실제 테스트에서 약 2,000줄 코드베이스도 우수하게 처리함이 확인되었습니다.

#### 오픈소스 리더십
MIT 라이선스로 공개되어 연구자와 개발자가 고급 기능을 자유롭게 활용할 수 있습니다. 기본 모델, 하이브리드 추론 버전, 최적화된 FP8 변형까지 제공합니다.

### 특별 기능

#### 에이전트 기능
GLM-4.5의 통합 아키텍처는 추론, 코딩, 도구 사용을 통합해야 하는 복잡한 지능형 에이전트 애플리케이션에 특히 적합합니다.

#### 포괄적 벤치마킹
성능 평가는 다음을 포함합니다:
- **에이전트 작업 벤치마크 3개**
- **추론 벤치마크 7개** 
- **코딩 벤치마크 2개**

이 종합 평가로 다양한 AI 애플리케이션에서의 다재다능함이 입증됩니다.

#### 개발자 통합
모델은 여러 프레임워크에서 통합을 지원합니다:
- **transformers**
- **vLLM**
- **SGLang**

모델 코드, 도구 파서, 추론 파서 구현이 함께 제공됩니다.

### 성능 비교

#### Claude 4 Sonnet 대비
GLM-4.5는 에이전트 코딩 및 추론 작업에서 경쟁력 있는 성능을 보이지만, Claude Sonnet 4는 코딩 성공률과 자율적 멀티 기능 애플리케이션 개발에서 우위를 유지합니다.

#### GPT-4.5 대비
GLM-4.5는 추론 및 에이전트 벤치마크에서 경쟁력 있는 순위를 보이며, GPT-4.5는 MMLU와 AIME 같은 전문 벤치마크에서 일반적으로 더 높은 정확도를 보입니다.

### 팁과 참고사항

-   **지역 선택:** 성능과 지역 규정 준수를 위해 적절한 지역을 선택하세요.
-   **모델 선택:** 최고 성능은 GLM-4.5, 효율성과 일반 하드웨어 호환성은 GLM-4.5-Air.
-   **컨텍스트 장점:** 128K 컨텍스트 창으로 대규모 코드와 문서를 처리할 수 있습니다.
-   **오픈소스 혜택:** MIT 라이선스는 상업적 사용과 2차 개발을 허용합니다.
-   **에이전트 애플리케이션:** 추론, 코딩, 도구 사용 통합이 필요한 애플리케이션에 특히 강합니다.
-   **하이브리드 추론:** 복잡한 문제는 Thinking Mode, 간단한 질의는 Non-Thinking Mode를 사용하세요.
-   **API 호환성:** OpenAI 호환 API로 스트리밍 응답 및 사용량 보고를 지원합니다.
-   **프레임워크 지원:** 배포 시나리오에 따라 여러 통합 옵션을 제공합니다.
