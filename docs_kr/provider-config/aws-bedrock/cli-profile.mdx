### 개요

Cline은 AWS Bedrock 서비스에 접근하기 위해 AWS 자격 증명 또는 AWS 프로필을 사용할 수 있습니다. 레거시 IAM 구성보다 SSO/연합 역할을 권장하며, 이 가이드는 Cline이 SSO 역할로 인증하도록 환경을 구성하는 방법을 설명합니다.

---

### 설정 단계

1. [AWS CLI 최신 버전](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)을 설치합니다.

    - AWS 문서를 따라 OS에 맞는 AWS CLI를 설치합니다.

2. AWS CLI로 [IAM 인증 구성](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html)을 합니다.

    - IAM Identity Center를 통해 아직 AWS 접근 권한이 없다면 [IAM 사용자 가이드](https://docs.aws.amazon.com/singlesignon/latest/userguide/getting-started.html)를 따라 IAM 사용자와 역할을 설정하세요. `PowerUserAccess` 역할이 필요합니다.
    - 직장에서 AWS 접근 권한이 있다면 AWS 액세스 포털을 열고 해당 계정을 찾으세요. `PowerUserAccess` 권한이 있는지 확인합니다.
    - `Access keys` 링크를 열고 `SSO start URL`과 `SSO region`을 기록합니다. 다음 단계에 필요합니다.

3. [ `aws configure sso` CLI 마법사](https://docs.aws.amazon.com/cli/latest/userguide/cli-configure-sso.html#cli-configure-sso-configure)로 프로필 구성을 계속합니다.

    - 구성이 끝나면 다음 명령으로 AWS CLI 인증을 수행합니다: `aws sso login --profile <AWS-profile-name>`
    - AWS 계정에 연결한 프로필 이름을 기록하세요. 다음 단계에서 Cline 설정에 필요합니다.

4. 아직 설치하지 않았다면 VSCode와 Cline 확장을 설치합니다. [시작하기](/getting-started) 페이지를 참고하세요.

5. Cline 확장을 열고 설정 버튼 ⚙️을 클릭해 API Provider를 선택합니다.
    - API Provider 드롭다운에서 AWS Bedrock을 선택합니다.
    - AWS Profile 라디오 버튼을 선택한 다음 3단계에서 기록한 AWS 프로필 이름을 입력합니다.
    - 드롭다운에서 AWS 리전을 선택합니다.
    - 일부 모델은 cross-region inference 체크박스가 필요합니다.
