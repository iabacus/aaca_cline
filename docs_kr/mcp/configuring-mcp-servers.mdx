## 전역 MCP 서버 포함 모드

MCP 서버를 사용하면 토큰 사용량이 늘어납니다. Cline은 MCP 서버 기능을 제한하거나 비활성화할 수 있는 옵션을 제공합니다.

1. Cline 확장 상단 네비게이션 바에서 "MCP Servers" 아이콘을 클릭합니다.
2. "Configure" 탭을 선택한 다음, 패널 하단의 "Advanced MCP Settings" 링크를 클릭합니다.
3. Cline이 새 설정 창을 열면 `Cline>Mcp:Mode`를 찾아 드롭다운에서 선택하세요.

## 개별 MCP 서버 관리

각 MCP 서버에는 자체 설정 패널이 있어 설정을 수정하고, 도구를 관리하며, 작동을 제어할 수 있습니다. 접근 방법은 다음과 같습니다:

1. Cline 확장 상단 네비게이션 바에서 "MCP Servers" 아이콘을 클릭합니다.
2. 목록에서 관리할 MCP 서버를 찾아 이름을 클릭해 엽니다.

	<img
		src="https://storage.googleapis.com/cline_public_images/docs/assets/MCP-settings-individual.png"

### 서버 삭제

1. 삭제할 MCP 서버 옆의 휴지통 아이콘을 클릭하거나, MCP 서버 설정 박스 하단의 빨간색 Delete Server 버튼을 클릭합니다.

**참고:** 삭제 확인 대화상자는 표시되지 않습니다.

### 서버 재시작

1. 재시작할 MCP 서버 옆의 Restart 버튼 또는 설정 박스 하단의 회색 Restart Server 버튼을 클릭합니다.

### 서버 활성화/비활성화

1. MCP 서버 옆의 토글 스위치로 서버를 개별적으로 켜고 끕니다.

### 네트워크 타임아웃

MCP 서버 도구 호출 후 응답을 기다리는 최대 시간을 설정하려면:

1. 개별 MCP 서버 설정 박스 하단의 `Network Timeout` 드롭다운을 클릭해 시간을 변경합니다. 기본값은 1분이며, 30초에서 1시간 사이로 설정할 수 있습니다.

## MCP 설정 파일 편집

설치된 모든 MCP 서버의 설정은 `cline_mcp_settings.json` 파일에 있습니다:

1. Cline 패널 상단 네비게이션 바에서 MCP Servers 아이콘 클릭
2. "Configure" 탭 선택
3. 패널 하단의 "Configure MCP Servers" 버튼 클릭

이 파일은 `mcpServers` 객체에 이름별 서버 구성을 담는 JSON 형식입니다:

```json
{
	"mcpServers": {
		"server1": {
			"command": "python",
			"args": ["/path/to/server.py"],
			"env": {
				"API_KEY": "your_api_key"
			},
			"alwaysAllow": ["tool1", "tool2"],
			"disabled": false
		}
	}
}
```

## 전송(Transport) 유형 이해하기

MCP는 서버 통신을 위해 두 가지 전송 유형을 지원합니다:

### STDIO Transport

로컬 머신에서 실행되는 서버에 사용됩니다:

-   표준 입력/출력 스트림으로 통신
-   낮은 지연(네트워크 오버헤드 없음)
-   보안성 향상(네트워크 노출 없음)
-   간단한 설정(HTTP 서버 불필요)
-   로컬에서 하위 프로세스로 실행

STDIO 전송이 동작하는 방식은 [MCP Transport Mechanisms](/mcp/mcp-transport-mechanisms)에서 자세히 확인하세요.

STDIO 구성 예시:

```json
{
	"mcpServers": {
		"local-server": {
			"command": "node",
			"args": ["/path/to/server.js"],
			"env": {
				"API_KEY": "your_api_key"
			},
			"alwaysAllow": ["tool1", "tool2"],
			"disabled": false
		}
	}
}
```

### SSE Transport

HTTP/HTTPS로 접근하는 원격 서버에 사용됩니다:

-   Server-Sent Events 프로토콜로 통신
-   다른 머신에 호스팅 가능
-   다중 클라이언트 연결 지원
-   네트워크 접근 필요
-   중앙 배포 및 관리 가능

SSE 전송이 동작하는 방식은 [MCP Transport Mechanisms](/mcp/mcp-transport-mechanisms)에서 자세히 확인하세요.

SSE 구성 예시:

```json
{
	"mcpServers": {
		"remote-server": {
			"url": "https://your-server-url.com/mcp",
			"headers": {
				"Authorization": "Bearer your-token"
			},
			"alwaysAllow": ["tool3"],
			"disabled": false
		}
	}
}
```

## 워크플로에서 MCP 도구 사용

MCP 서버를 구성하면 Cline이 사용 가능한 도구와 리소스를 자동으로 감지합니다. 사용 방법은 다음과 같습니다:

1. Cline 대화창에 요청 입력
2. Cline이 작업에 도움이 되는 MCP 도구를 식별
3. 요청 시 도구 사용 승인(또는 자동 승인 사용)

예: "API 성능을 분석해줘"는 API 엔드포인트를 테스트하는 MCP 도구를 사용할 수 있습니다.

## MCP 서버 문제 해결

일반적인 문제와 해결 방법:

-   **서버 응답 없음:** 서버 프로세스가 실행 중인지 확인하고 네트워크 연결을 점검
-   **권한 오류:** `mcp_settings.json`에 올바른 API 키와 자격 증명이 설정되어 있는지 확인
-   **도구 없음:** 서버가 도구를 올바르게 구현했는지, 설정에서 비활성화되지 않았는지 확인
-   **느린 성능:** 해당 MCP 서버의 네트워크 타임아웃 값을 조정
