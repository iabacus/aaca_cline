---
title: "GitHub에서 MCP 서버 추가하기"
---

이 가이드는 GitHub 저장소에 있는 기존 MCP 서버를 Cline으로 빌드하는 방법을 단계별로 안내합니다.

## **MCP 서버 찾기**

MCP 서버를 찾을 수 있는 곳은 여러 곳입니다:

-   **Cline이 자동으로 MCP 서버를 목록에 추가하고 편집할 수 있습니다.** Cline은 GitHub에서 저장소를 바로 클론하고 서버를 빌드해 줍니다.
-   **GitHub:** MCP 서버를 찾기 좋은 대표 저장소 두 곳은 다음과 같습니다:
    -   [공식 MCP 서버 저장소](https://github.com/modelcontextprotocol/servers)
    -   [Awesome-MCP 서버 저장소](https://github.com/punkpeye/awesome-mcp-servers)
-   **온라인 디렉터리:** 다양한 웹사이트에 MCP 서버 목록이 있습니다:

    -   [mcpservers.org](https://mcpservers.org/)
    -   [mcp.so](https://mcp.so/)
    -   [glama.ai/mcp/servers](https://glama.ai/mcp/servers)
    -   [mcp.composio.dev](https://mcp.composio.dev/)

    이 디렉터리들은 다운로드, 날짜, 스타, 사용 사례 등 다양한 기준으로 서버를 정렬할 수 있습니다. 각 항목은 기능, 도구, 구성 지침 등을 제공합니다.

-   **PulseMCP:** AI가 MCP 서버를 사용해 웹사이트를 대체할 수 있다는 논의를 담은 블로그 글과 MCP 서버 FAQ를 제공하는 사이트입니다: [https://www.pulsemcp.com/](https://www.pulsemcp.com/)

## **Cline으로 빌드하기**

1. **프로세스 시작:** Cline에 다음 정보를 제공합니다:
    - **GitHub 저장소 URL:** 서버 저장소의 URL
    - **README.md 내용:** 선택 사항이지만 서버의 목적과 구성 이해에 도움이 됩니다. GitHub 저장소에서 README.md를 복사하세요.
2. **Cline과의 예시 대화:**

    ```
    User: "Cline, Brave 브라우저 제어용 MCP 서버를 추가하고 싶어. GitHub 링크는 @https://github.com/modelcontextprotocol/servers/tree/main/src/brave-search 야. 추가해줄 수 있어?"

    Cline: "OK. MCP 디렉터리로 저장소를 클론합니다. 'package.json'이 있으니 빌드가 필요합니다. 'npm run build'를 실행할까요?"

    User: "응, 빌드해줘."

    Cline: "빌드 완료. 이 서버는 브라우저 경로가 필요합니다. Brave 브라우저 실행 파일 경로는 어디에 있나요?"

    User: "'/Applications/Brave Browser.app/Contents/MacOS/Brave Browser'에 있어."

    Cline: "OK. 해당 경로로 서버 설정을 업데이트하고 'cline_mcp_settings.json'에 추가했습니다. 다른 건 더 필요할까요?"

    User: "아니, 됐어. 테스트하자."

    Cline: "좋아요! MCP Inspector로 서버 연결을 테스트하겠습니다. 그 다음 Cline에서 브라우저를 제어해볼 수 있습니다."
    ```

3. **Cline의 작업:** 지시 내용에 따라 Cline이 다음을 수행합니다:
    - **저장소 클론:** 보통 설정에 지정된 디렉터리에 저장소를 클론합니다.
    - **조정:** 서버 설정 수정이 필요할 때 Cline에게 안내할 수 있습니다. 예:
        - **User:** "이 서버에 API 키가 필요해. 어디에 추가해야 하는지 찾아줄래?"
        - Cline이 `cline_mcp_settings.json` 또는 관련 파일을 자동으로 수정할 수 있습니다.
    - **서버 빌드:** 일반적으로 `npm run build` 같은 빌드 명령 실행
    - **설정에 서버 추가:** `cline_mcp_settings.json`에 서버 구성을 추가

## **테스트 및 문제 해결**

1. **서버 테스트:** 빌드가 끝나면 서버가 정상 동작하는지 테스트하세요. 문제가 있으면 Cline이 도와줄 수 있습니다.
2. **MCP Inspector:** MCP Inspector로 서버 연결과 기능을 테스트할 수 있습니다.

## **모범 사례**

-   **기본 이해:** Cline이 과정을 단순화하더라도, 서버 코드, MCP 프로토콜([자세히 보기](/mcp/mcp-overview)), 구성 방법을 기본적으로 이해하면 문제 해결과 커스터마이징에 도움이 됩니다.
-   **명확한 지침:** 과정 전반에서 Cline에 구체적이고 명확한 지시를 제공하세요.
-   **테스트:** 설치 및 구성 후 서버가 올바르게 동작하는지 충분히 테스트하세요.
-   **버전 관리:** 서버 코드 변경을 추적하기 위해 Git 같은 버전 관리 시스템을 사용하세요.
-   **업데이트 유지:** 최신 기능과 보안 패치를 받기 위해 MCP 서버를 최신 상태로 유지하세요.
