## MCP 서버란?

MCP 서버는 Cline의 기능을 확장하는 특화 확장입니다. 웹페이지 가져오기, 이미지 처리, API 접근 등 다양한 작업을 수행할 수 있게 해줍니다.

## MCP Marketplace 사용 가이드

MCP Marketplace는 다양한 카테고리의 수백 개 MCP 서버를 원클릭으로 설치할 수 있는 경험을 제공합니다.

### 1. 마켓플레이스 접근

-   Cline 상단 툴바에서 "Extensions" 버튼(사각형 아이콘) 클릭
-   MCP 마켓플레이스가 열리며 카테고리별 서버가 표시됩니다

### 2. 서버 탐색 및 선택

-   카테고리별로 서버 탐색(Search, File-systems, Browser-automation, Research-data 등)
-   서버를 클릭해 기능과 요구사항을 확인

### 3. 설치 및 구성

-   원하는 서버의 설치 버튼 클릭
-   API 키가 필요한 경우(대부분) Cline이 다음을 안내합니다:
    -   API 키 발급 위치
    -   안전하게 입력하는 방법
-   서버가 MCP 설정에 자동 추가됩니다

### 4. 설치 확인

-   Cline이 설치 완료 메시지를 표시
-   Cline의 MCP 설정 UI에서 서버 상태 확인

### 5. 새 서버 사용

-   설치가 완료되면 Cline이 서버 기능을 자동 통합
-   Cline의 시스템 프롬프트에 새 도구와 리소스가 표시됨
-   Cline에게 해당 서버 기능을 사용하도록 요청
-   예: "Perplexity로 최신 React 업데이트를 검색해줘"

**기업 사용자:** 기업 환경에서 Cline을 사용 중이라면, 조직 보안 정책에 따라 서드파티 MCP 서버 설치 권한이 있는지 확인하세요.

## 내부적으로 무엇이 일어나는가

MCP 서버를 설치하면 다음이 자동으로 수행됩니다:

### 1. 설치 프로세스

-   서버 코드가 `/Users/<username>/Documents/Cline/MCP/`에 클론/설치됨
-   의존성 설치
-   서버 빌드(TypeScript/JavaScript 컴파일 또는 Python 패키지 설치)

### 2. 구성

-   MCP 설정 파일이 서버 구성으로 업데이트됨
-   파일 경로: `/Users/<username>/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json`
-   환경 변수(API 키 등)가 안전하게 저장됨
-   서버 경로가 등록됨

### 3. 서버 실행

-   Cline이 설정 변경을 감지
-   서버를 별도 프로세스로 실행
-   stdio 또는 HTTP로 통신 연결

### 4. Cline 통합

-   서버 기능이 Cline 시스템 프롬프트에 추가됨
-   `use_mcp_tool` 명령으로 도구 사용 가능
-   `access_mcp_resource` 명령으로 리소스 접근 가능
-   사용자의 요청에 따라 Cline이 해당 기능을 사용

## 문제 해결

### 시스템 요구사항

다음 요구사항을 충족하는지 확인하세요:

-   **Node.js 18.x 이상**
    -   확인: `node --version`
    -   설치: https://nodejs.org/
    -   JavaScript/TypeScript 구현에 필요
-   **Python 3.10 이상**
    -   확인: `python --version`
    -   설치: https://python.org/
    -   참고: 일부 구현은 Python 3.11+ 필요
-   **UV 패키지 매니저**
    -   의존성 격리를 위한 최신 Python 패키지 매니저
    -   설치:
        ```bash
        curl -LsSf https://astral.sh/uv/install.sh | sh
        ```
        또는: `pip install uv`
    -   확인: `uv --version`

이 명령들이 실패하거나 버전이 낮다면 설치/업데이트 후 계속하세요.

### 일반적인 설치 문제

-   인터넷 연결이 안정적인지 확인
-   새 소프트웨어 설치 권한이 있는지 확인
-   API 키가 올바르게 입력됐는지 확인(필요 시)
-   MCP 설정 UI에서 서버 상태와 오류 메시지 확인

### MCP 서버 제거 방법

문제가 있는 MCP 서버를 완전히 제거하려면:

1. MCP 설정 파일 열기: `/Users/<username>/Library/Application Support/Code/User/globalStorage/saoudrizwan.claude-dev/settings/cline_mcp_settings.json`
2. `mcpServers` 객체에서 해당 서버 항목 전체 삭제
3. 파일 저장
4. Cline 재시작

### 오류가 계속 발생할 때

MCP 서버 사용 중 오류가 발생한다면 다음을 시도하세요:

-   MCP 설정 파일에 오류가 없는지 확인
-   설치 시 Claude Sonnet 모델 사용
-   서버 파일 경로가 정확한지 확인
-   필요한 환경 변수가 모두 설정되어 있는지 확인
-   같은 포트를 사용하는 다른 프로세스가 있는지 확인(HTTP 기반 서버)
-   서버 제거 후 재설치(`cline_mcp_settings.json` 파일과 `/Users/<username>/Documents/Cline/MCP/` 디렉터리 모두에서 제거)
-   터미널에서 동일한 명령과 인자를 직접 실행해 Cline이 보는 오류를 확인

## MCP 서버 규칙

Cline은 활성 MCP 서버와 역할을 알고 있지만, 서버가 많아지면 각 서버를 언제 사용할지 정의하는 것이 유용합니다.

`.clinerules` 파일이나 커스텀 지침을 사용해 키워드 기반 트리거로 MCP 서버를 지능적으로 활성화하면, 도구 선택이 더 직관적이고 컨텍스트에 맞게 됩니다.

### MCP 규칙 동작 방식

MCP 규칙은 연결된 MCP 서버를 기능 카테고리로 그룹화하고, 대화 중 키워드가 감지되면 자동 활성화되도록 트리거를 정의합니다.

```json
{
	"mcpRules": {
		"webInteraction": {
			"servers": ["firecrawl-mcp-server", "fetch-mcp"],
			"triggers": ["web", "scrape", "browse", "website"],
			"description": "Tools for web browsing and scraping"
		}
	}
}
```

### 구성 구조

1. **카테고리**: 관련 서버를 그룹화(예: "webInteraction", "mediaAndDesign")
2. **서버**: 각 카테고리의 서버 이름 목록
3. **트리거**: 해당 서버를 활성화하는 키워드
4. **설명**: 카테고리 설명(사람이 읽기 쉬운 문장)

### MCP 규칙의 이점

-   **컨텍스트 기반 도구 선택**: 대화 컨텍스트에 맞는 도구 선택
-   **마찰 감소**: 어떤 도구를 쓸지 수동으로 지정할 필요 없음
-   **능력 정리**: 관련 도구와 서버를 논리적으로 그룹화
-   **우선순위 부여**: 모호한 경우 명시적 우선순위 처리

### 사용 예시

"이 웹사이트를 스크래핑해줘"라고 쓰면, "scrape"와 "website" 트리거가 감지되어 웹 관련 MCP 서버가 자동 선택됩니다.

"애플 주가가 얼마야?" 같은 금융 요청은 "stock", "price" 같은 키워드가 감지되어 금융 관련 서버를 트리거합니다.

### 빠른 시작 템플릿

```json
{
	"mcpRules": {
		"category1": {
			"servers": ["server-name-1", "server-name-2"],
			"triggers": ["keyword1", "keyword2", "phrase1", "phrase2"],
			"description": "Description of what these tools do"
		},
		"category2": {
			"servers": ["server-name-3"],
			"triggers": ["keyword3", "keyword4", "phrase3"],
			"description": "Description of what these tools do"
		},
		"category3": {
			"servers": ["server-name-4", "server-name-5"],
			"triggers": ["keyword5", "keyword6", "phrase4"],
			"description": "Description of what these tools do"
		}
	},
	"defaultBehavior": {
		"priorityOrder": ["category1", "category2", "category3"],
		"fallbackBehavior": "Ask user which tool would be most appropriate"
	}
}
```

이 템플릿을 `.clinerules` 파일이나 커스텀 지침에 추가하면 Cline의 MCP 서버 선택이 더 직관적이고 컨텍스트에 맞게 동작합니다.
