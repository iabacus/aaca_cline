## 컨텍스트 윈도우란?

컨텍스트 윈도우는 AI 모델이 한 번에 처리할 수 있는 최대 텍스트 양입니다. 모델의 "작업 메모리"라고 생각하면 됩니다. 대화와 코드 중 얼마나 많은 내용을 고려할 수 있는지 결정합니다.

<Note>
**핵심 포인트**: 컨텍스트 윈도우가 클수록 더 많은 코드베이스를 한 번에 이해할 수 있지만, 비용과 응답 시간이 증가할 수 있습니다.
</Note>

## 컨텍스트 효율적으로 관리하기

### 컨텍스트에 포함되는 것

1. **현재 대화** - 채팅의 모든 메시지
2. **파일 내용** - 공유했거나 Cline이 읽은 파일
3. **도구 출력** - 실행된 명령 결과
4. **시스템 프롬프트** - Cline의 지침(영향은 작음)

### 최적화 전략

#### 1. 새 기능은 새롭게 시작
```
/new - 깨끗한 컨텍스트로 새 작업 생성
```
장점:
- 최대 컨텍스트 확보
- 불필요한 히스토리 없음
- 모델 집중도 향상

#### 2. @ 멘션을 전략적으로 사용
전체 파일을 넣는 대신:
- `@filename.ts` - 필요할 때만 포함
- 큰 파일은 검색으로 대체
- 파일 전체가 아닌 특정 함수 참조

#### 3. 자동 압축 활성화
Cline이 긴 대화를 자동 요약할 수 있습니다:
- Settings → Features → Auto-compact
- 중요한 컨텍스트 유지
- 토큰 사용량 감소

## 컨텍스트 윈도우 경고

### 한도에 도달하는 징후

| 경고 신호 | 의미 | 해결 방법 |
|-------------|---------------|----------|
| **"Context window exceeded"** | 하드 한도 도달 | 새 작업 시작 또는 자동 압축 활성화 |
| **응답 느려짐** | 컨텍스트 부담 증가 | 포함 파일 줄이기 |
| **반복 제안** | 컨텍스트 단편화 | 요약 후 새로 시작 |
| **최근 변경 누락** | 컨텍스트 오버플로 | 체크포인트로 변경 추적 |

### 프로젝트 규모별 모범 사례

#### 작은 프로젝트(< 50 파일)
- 어떤 모델이든 잘 작동
- 관련 파일 자유롭게 포함
- 특별한 최적화 불필요

#### 중간 프로젝트(50-500 파일)
- 128K+ 컨텍스트 모델 사용
- 작업 중인 파일 세트만 포함
- 기능 단위로 컨텍스트 정리

#### 큰 프로젝트(500+ 파일)
- 200K+ 컨텍스트 모델 사용
- 특정 모듈에 집중
- 많은 파일 읽기 대신 검색 사용
- 작업을 작은 단위로 분할

## 고급 컨텍스트 관리

### Plan/Act 모드 최적화

Plan/Act 모드를 활용해 컨텍스트를 효율적으로 사용하세요:
- **Plan 모드**: 토론용으로 작은 컨텍스트 사용
- **Act 모드**: 구현에 필요한 파일 포함

구성 예:
```
Plan 모드: DeepSeek V3 (128K) - 저비용 계획
Act 모드: Claude Sonnet (1M) - 최대 컨텍스트 코딩
```

### 컨텍스트 가지치기 전략

1. **시간 기반 가지치기**: 오래된 대화 부분 제거
2. **의미 기반 가지치기**: 관련 코드 부분만 유지
3. **계층적 가지치기**: 상위 구조는 유지하고 세부는 제거

### 토큰 계산 팁

#### 대략적인 추정
- **1 토큰 ≈ 0.75 단어**
- **1 토큰 ≈ 4 글자**
- **코드 100줄 ≈ 500-1000 토큰**

#### 파일 크기 가이드
| 파일 유형 | KB당 토큰 |
|-----------|---------------|
| **코드** | ~250-400 |
| **JSON** | ~300-500 |
| **Markdown** | ~200-300 |
| **일반 텍스트** | ~200-250 |

## 컨텍스트 윈도우 FAQ

### Q: 대화가 길어질수록 왜 응답 품질이 떨어지나요?
**A:** 모델은 컨텍스트가 너무 많으면 집중을 잃을 수 있습니다. "유효 윈도우"는 보통 광고된 한도의 50~70% 정도입니다.

### Q: 가능한 가장 큰 컨텍스트 윈도우를 써야 하나요?
**A:** 항상 그렇진 않습니다. 큰 컨텍스트는 비용을 올리고 응답 품질을 떨어뜨릴 수 있습니다. 작업 규모에 맞는 크기를 선택하세요.

### Q: 내가 얼마나 컨텍스트를 쓰는지 어떻게 알 수 있나요?
**A:** Cline 인터페이스에서 토큰 사용량을 보여줍니다. 컨텍스트 미터가 한도에 가까워지는지 확인하세요.

### Q: 컨텍스트 한도를 넘으면 어떻게 되나요?
**A:** Cline은 다음 중 하나를 수행합니다:
- 자동으로 대화를 압축(활성화된 경우)
- 오류를 표시하고 새 작업 시작을 제안
- 오래된 메시지를 잘라냄(경고 포함)
