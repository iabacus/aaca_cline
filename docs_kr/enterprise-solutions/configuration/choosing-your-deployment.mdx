조직에 맞는 구성 접근 방식을 선택하세요. 대부분의 팀은 빠른 배포를 위해 SaaS로 시작하며, 복잡한 요구사항이 있는 엔터프라이즈는 자체 호스팅 인프라를 선택합니다.

    - 기존 클라우드 인프라 보유
    - 멀티 리전 배포
    - 맞춤형 컴플라이언스 요구

## 상세 비교

### 기능 비교

| 기능 | SaaS | 자체 호스팅 |
|---------|------|-------------|
| **구성 방식** | 웹 UI | YAML + Helm/Kubernetes |
| **인프라** | 필요 없음 | 전체 AWS/GCP/K8s |
| **VPC 엔드포인트** | 기본 | 완전한 프라이빗 연결 |
| **다중 계정** | ❌ | ✅ |
| **IAM** | 표준 RBAC 역할 | 표준 RBAC 역할 |
| **컴플라이언스** | 표준 | 맞춤 프레임워크 |
| **GitOps** | ❌ | ✅ |
| **유지보수** | Cline에서 관리 | 자체 관리 |
| **업데이트** | 자동(확장) | 자동(확장) + 인프라 제어 |

### 보안 및 컴플라이언스

| 역량 | SaaS | 자체 호스팅 |
|------------|------|-------------|
| **네트워크 암호화** | HTTPS/TLS | HTTPS/TLS |
| **네트워크** | 공용 인터넷 | 프라이빗 VPC 엔드포인트 |
| **접근 제어** | 표준 RBAC | 표준 RBAC |
| **감사 로그** | OpenTelemetry 트레이스 | OpenTelemetry 트레이스 + 인프라 로그 |
| **데이터 레지던시** | Cline 관리 배포 | 고객 제어 배포 |

### 비용 구조

| 비용 항목 | SaaS | 자체 호스팅 |
|---------------|------|-------------|
| **Cline 구독** | 고정 엔터프라이즈 요금 | 고정 엔터프라이즈 요금 |
| **추론 제공자 비용** | 사용량 기반 | 사용량 기반 |
| **인프라** | ✅ 필요 없음 | Kubernetes, 네트워킹, 스토리지 |
| **인력** | ✅ 필요 없음 | DevOps 팀 필요 |
| **총비용 프로필** | 예측 가능하고 단순함 | 규모에 따라 변동 |

## 마이그레이션 경로

<Note>
대부분의 조직은 빠른 배포를 위해 SaaS 구성으로 시작한 뒤, 요구사항이 커지면 자체 호스팅으로 전환합니다. 이는 위험을 최소화하고 실제 사용 패턴에 맞는 인프라를 구축하는 데 도움이 됩니다.
</Note>
