팀원으로서 로컬 개발 환경을 조직의 Google Vertex AI 구성에 연결할 수 있습니다. 이 가이드는 VS Code에서 Google Cloud 자격 증명을 설정해 조직의 프로젝트와 리전 설정으로 Vertex AI 모델을 사용할 수 있도록 안내합니다. 제공자 설정은 이미 관리자가 구성했으므로, 시작을 위해 자격 증명만 추가하면 됩니다.

## 시작하기 전에

조직의 Google Vertex AI 설정에 성공적으로 연결하려면 다음을 준비해야 합니다.

**Cline 확장 설치 및 구성**  
VS Code에 Cline 확장이 설치되어 있어야 하며, 조직 계정으로 로그인해야 합니다. 아직 설치하지 않았다면 [설치 가이드](/getting-started/installing-cline)를 따라주세요.

<Info>
**빠른 확인**: VS Code의 Cline 패널을 열고 좌하단에 조직명이 보이면 정상 로그인 상태입니다.
</Info>

**Vertex AI 접근 권한이 있는 Google Cloud 자격 증명**  
조직에 설정된 프로젝트와 리전에서 Vertex AI 접근 권한이 있는 Google Cloud 자격 증명이 필요합니다.

<Note>
어떤 방법을 사용해야 할지 모르겠다면, 조직의 Google Cloud 접근 설정 방식에 대해 관리자나 IT 팀에 문의하세요.
</Note>

## 구성 단계

<Steps>
<Step title="Cline 설정 열기">
VS Code에서 다음 중 하나의 방법으로 Cline 설정 패널을 엽니다:

- Cline 패널의 설정 아이콘(⚙️) 클릭
- 채팅 영역 바로 아래의 API Provider 드롭다운 클릭(예: `vertex_ai/gemini-pro` 또는 유사)

</Step>

<Step title="인증 방식 선택">
Google Vertex AI 인증을 위한 다음 자격 증명 방식 중 하나를 선택하세요:

<AccordionGroup>
<Accordion title="서비스 계정 키">
Vertex AI 접근용 서비스 계정 JSON 키 파일을 사용합니다.

[서비스 계정 키 자세히 보기](https://cloud.google.com/iam/docs/service-accounts)

1. **Service Account Key** 인증 방식 선택
2. 서비스 계정 JSON 키 내용을 업로드하거나 붙여넣기
3. 키에 `aiplatform.user` 또는 유사한 Vertex AI 권한이 있어야 합니다.
4. 자격 증명은 로컬에 저장되며 VS Code 확장만 사용합니다.
</Accordion>

<Accordion title="Google Cloud SDK">
컴퓨터에 설치된 Google Cloud SDK와 인증된 계정을 사용합니다.

[Google Cloud SDK 자세히 보기](https://cloud.google.com/sdk/docs/install)

1. **Google Cloud SDK** 인증 방식 선택
2. `gcloud auth login`으로 인증했는지 확인
3. 조직의 Vertex AI 프로젝트에 접근 권한이 있는지 확인
4. Cline이 기본 Google Cloud 자격 증명을 자동 사용
</Accordion>

<Accordion title="애플리케이션 기본 자격 증명">
Google Cloud의 ADC(Application Default Credentials) 체인을 사용합니다.

1. **Application Default Credentials** 방식 선택
2. 환경에서 ADC가 올바르게 구성되어 있는지 확인
3. 조직에서 중앙으로 자격 증명을 관리하는 환경에 적합
4. Cline이 환경에서 자동으로 자격 증명을 감지
</Accordion>
</AccordionGroup>

<Note>
Google Cloud 프로젝트 ID와 리전은 관리자가 미리 구성했으므로 확장에서 설정할 필요가 없습니다.
</Note>
</Step>

<Step title="구성 확인">
인증 방식을 선택하면 확장에서 활성화된 기능에 대한 체크 표시가 나타납니다:

- ✓ 이미지 지원(Gemini Pro Vision 등)
- ✓ 멀티모달 입력 지원
- ✓ 함수 호출 지원(지원 모델)

프로젝트 ID와 리전 설정은 관리자에 의해 제어되므로 잠금 아이콘(🔒)으로 표시됩니다.
</Step>

<Step title="연결 테스트">
Cline에서 테스트 메시지를 보내 구성된 Vertex AI 프로젝트/리전에서 자격 증명이 정상 동작하는지 확인합니다.

<Tip>
**테스트 권장**

먼저 "Hello" 같은 간단한 테스트로 기본 연결을 확인하고, 필요하면 이미지를 공유해 멀티모달 기능도 테스트하세요.
</Tip>
</Step>
</Steps>

## 모델 사용

### 사용 가능한 모델 패밀리

조직의 Vertex AI 구성에서 일반적으로 다음 모델을 사용할 수 있습니다:

**Gemini 모델:**
- **Gemini Pro**: 고급 추론, 코드 생성, 멀티모달 기능
- **Gemini Pro Vision**: 이미지 이해 및 시각적 질문 응답
- **Gemini Ultra**: 복잡한 추론 작업용 최상위 모델

**PaLM 모델:**
- **PaLM 2 for Text**: 텍스트 생성 및 완성
- **PaLM 2 for Chat**: 대화형 AI 상호작용
- **Codey**: 코드 생성/설명에 특화

**특수 모델:**
- **Text Embedding**: 시맨틱 검색 및 유사도 작업
- **Custom Models**: 조직의 파인튜닝 모델(가능한 경우)

### 모델 선택 전략

개발 요구에 따라 모델을 선택하세요:

- **일반 작업**: 대부분의 텍스트/추론 작업에 Gemini Pro
- **시각 콘텐츠**: 이미지 작업은 Gemini Pro Vision
- **코드 중심 작업**: 프로그래밍 작업은 Codey 모델
- **복잡한 추론**: Gemini Ultra
- **임베딩 작업**: Text Embedding 모델

### 멀티모달 기능

Vertex AI의 멀티모달 기능을 활용하세요:

- **이미지 분석**: Cline에 이미지를 직접 업로드해 분석
- **시각적 질문 응답**: 이미지에 대한 질문
- **코드 스크린샷**: 스크린샷에서 코드 설명 받기
- **문서 처리**: 차트, 그래프 등 시각적 데이터 분석

## 문제 해결

**Google Vertex AI가 제공자 옵션에 없음**  
올바른 Cline 조직에 로그인했는지 확인하세요. 관리자가 Vertex AI 구성을 저장했는지 확인하고 Cline 확장이 최신 버전인지 확인하세요.

**인증 오류("Access Denied" 또는 "Invalid Credentials")**  
선택한 자격 증명 방식에 구성된 프로젝트/리전에서 Vertex AI 접근 권한이 있는지 확인하세요. 필요한 권한에는 `aiplatform.endpoints.predict`, `aiplatform.models.predict`가 포함됩니다.

**프로젝트 접근 오류**  
관리자에게 조직에 설정된 Google Cloud 프로젝트를 확인해 달라고 요청하세요. 해당 프로젝트에 접근 권한이 있는지 확인하세요.

**리전 접근 오류**  
구성된 리전에서 Vertex AI 접근 권한이 있는지 확인하세요. 일부 모델은 모든 리전에서 제공되지 않을 수 있으니 관리자에게 선택된 리전을 확인하세요.

**Google Cloud SDK 인증 문제**  
Google Cloud SDK가 올바르게 설치/인증되었는지 확인하세요:
```bash
gcloud auth login
gcloud config set project YOUR_PROJECT_ID
gcloud auth application-default login
```

**서비스 계정 키 오류**  
서비스 계정 키가 유효하고 만료되지 않았는지 확인하세요. 조직 프로젝트에서 올바른 Vertex AI 권한이 있는지도 확인하세요. JSON 키 파일이 올바르게 포맷되어 있는지 확인하세요.

**모델 접근 오류 또는 "model not found"**  
일부 모델은 조직의 프로젝트/리전에서 활성화되지 않았을 수 있습니다. 특정 모델이 보이지 않으면 관리자에게 문의하세요. Google Cloud Console에서 사용하려는 모델이 활성화되어 있는지 확인하세요.

## 보안 모범 사례

Google Cloud 자격 증명을 구성할 때 다음 보안 가이드를 따르세요:

- Vertex AI 접근에 필요한 최소 권한의 서비스 계정 사용
- 서비스 계정 키를 정기적으로 로테이션(권장: 90일)
- 자격 증명을 코드나 버전 관리에 저장하지 않기
- 가능한 경우 Google Cloud SDK 사용으로 자격 증명 관리 개선
- 컨테이너 개발 환경에는 Workload Identity 사용 고려
- 의심스러운 활동이나 무단 접근 시 즉시 보고

조직 관리자가 사용 가능한 모델과 리전을 제어합니다. 확장은 프로젝트 구성과 리전 가용성에 따라 사용 가능한 모델을 자동으로 표시합니다.

Google Cloud 인증 및 Vertex AI 권한에 대한 자세한 내용은 [Google Cloud IAM 문서](https://cloud.google.com/iam/docs)를 참고하고, 조직의 클라우드 관리자와 협력하세요.
