Cline은 조직의 추론 제공자와 시스템 설정을 구성하기 위한 두 가지 접근 방식을 제공합니다. 각 방식의 차이를 이해하면 요구사항에 맞는 구성을 선택할 수 있습니다.

## 올바른 구성 선택하기

### SaaS 구성을 사용할 때
- **빠른 설정**: 팀을 신속히 시작해야 할 때
- **중앙 관리**: 간단한 클라우드 기반 제공자 관리를 원할 때
- **표준 요구사항**: 일반적인 보안/컴플라이언스 요구 수준
- **소~중규모 팀**: 수십~수백 명 사용자 관리

### 자체 호스팅 구성을 사용할 때
- **엔터프라이즈 보안**: 고급 보안 기능과 컴플라이언스 통제가 필요할 때
- **복잡한 인프라**: 기존 AWS/GCP 인프라와 통합해야 할 때
- **커스텀 워크플로**: 규칙, 워크플로, 자동화가 필요할 때
- **대규모 조직**: 수백~수천 명 사용자 관리
- **망 분리 환경**: 온프레미스 또는 제한된 네트워크 배포가 필요할 때

## 구성 비교

| 기능 | SaaS 구성 | 자체 호스팅 구성 |
|---------|-------------------|---------------------------|
| **설정 난이도** | 쉬움 | 고급 |
| **배포 시간** | 수 분 | 수일~수주 |
| **필요 인프라** | 없음 | AWS/GCP/Azure |
| **컴플라이언스 기능** | 기본 | 고급 |
| **커스텀 규칙** | 아니오 | 예 |
| **다중 계정 지원** | 아니오 | 예 |
| **VPC 통합** | 아니오 | 예 |
| **비용** | 낮음 | 높음 |

## 시작하기

<Steps>
<Step title="요구사항 평가">
조직의 보안, 컴플라이언스, 인프라 요구사항을 검토하여 적합한 구성 방식을 결정하세요.
</Step>

<Step title="경로 선택">
간단한 설정은 SaaS 구성, 고급 엔터프라이즈 기능은 자체 호스팅 구성을 선택하세요. 필요하면 [배포 가이드](/enterprise-solutions/configuration/choosing-your-deployment)를 참고하세요.
</Step>

<Step title="구성 가이드 따라하기">
선택한 구성 유형에 대한 상세 가이드를 따라 설정을 완료합니다.
</Step>

<Step title="팀 온보딩">
구성이 완료되면 팀원들은 제공자별 멤버 가이드를 통해 연결할 수 있습니다.
</Step>
</Steps>
