<Info>
**구성 경로: 자체 호스팅**

이 가이드는 자체 호스팅 배포를 위한 Vertex AI 구성을 다룹니다. 웹 기반 간단 설정은 [Google Vertex SaaS 구성](/enterprise-solutions/configuration/remote-configuration/google-vertex/admin-configuration)을 참고하세요.
</Info>

Google Cloud Platform을 통해 Gemini 및 기타 Google AI 모델을 엔터프라이즈에서 사용할 수 있도록 Cline에 Google Vertex AI를 구성합니다.

## 구성 형식

원격 구성 JSON에서 `providerSettings.Vertex` 섹션으로 Vertex AI를 구성합니다:

```json
{
  "providerSettings": {
    "Vertex": {
      "models": [
        {
          "id": "claude-3-5-sonnet-v2@20241022",
          "name": "Claude 3.5 Sonnet"
        }
      ],
      "vertexProjectId": "my-project-id",
      "vertexRegion": "us-central1"
    }
  }
}
```

## 구성 필드

| 필드 | 타입 | 설명 | 필수 |
|-------|------|-------------|----------|
| `models` | Array | 모델 구성 목록 | 예 |
| `vertexProjectId` | String | Google Cloud 프로젝트 ID | 예 |
| `vertexRegion` | String | GCP 리전(예: `us-central1`) | 예 |

### 모델 구성

`models` 배열의 각 모델은 다음을 포함해야 합니다:

```json
{
  "id": "claude-3-5-sonnet-v2@20241022",
  "name": "Claude 3.5 Sonnet",
  "info": {
    "maxTokens": 8192,
    "contextWindow": 200000,
    "supportsImages": true,
    "supportsPromptCache": true
  }
}
```

## 일반적인 모델 ID

| 모델 ID | 설명 | 컨텍스트 윈도우 |
|----------|-------------|----------------|
| `claude-3-5-sonnet-v2@20241022` | Claude 3.5 Sonnet | 200K 토큰 |
| `claude-3-5-haiku@20241022` | Claude 3.5 Haiku | 200K 토큰 |
| `claude-3-opus@20240229` | Claude 3 Opus | 200K 토큰 |
| `gemini-2.0-flash-exp` | Gemini Flash(실험적) | 1M 토큰 |
| `gemini-1.5-pro-002` | Gemini Pro | 2M 토큰 |
| `gemini-1.5-flash-002` | Gemini Flash | 1M 토큰 |

<Note>
모델 가용성은 리전에 따라 다릅니다. 리전별 모델 가용성은 [Vertex AI 문서](https://cloud.google.com/vertex-ai/docs/generative-ai/learn/models)를 참고하세요.
</Note>

## 사전 준비

Vertex AI를 사용하기 전에 다음이 필요합니다:

1. Vertex AI API가 활성화된 **Google Cloud 프로젝트**
2. Vertex AI User 역할(`roles/aiplatform.user`)을 가진 **서비스 계정**
3. **서비스 계정 자격 증명** 구성
4. 프로젝트 및 리전에서 **모델 접근** 확인

<Tip>
Google Cloud 설정과 인증 구성은 [Vertex AI 문서](https://cloud.google.com/vertex-ai/docs/generative-ai/start/quickstarts/quickstart-multimodal)를 참고하세요.
</Tip>

## 문제 해결

**"Permission Denied" 오류**

서비스 계정에 필요한 Vertex AI 권한이 있는지 확인하세요. 권한 요구사항은 [Google Cloud IAM 문서](https://cloud.google.com/vertex-ai/docs/general/access-control)를 참고하세요.

**"API Not Enabled" 오류**

Google Cloud 프로젝트에서 Vertex AI API가 활성화되어 있는지 확인하세요.

**"Model Not Found" 오류**

구성한 리전에서 모델이 उपलब्ध하고 프로젝트에 접근 권한이 있는지 확인하세요.
