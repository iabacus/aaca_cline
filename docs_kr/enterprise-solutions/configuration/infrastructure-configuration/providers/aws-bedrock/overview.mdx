<Info>
**구성 경로: 자체 호스팅**

이 가이드는 자체 호스팅 배포를 위한 Bedrock 구성을 다룹니다. 웹 기반 간단 설정은 [AWS Bedrock SaaS 구성](/enterprise-solutions/configuration/remote-configuration/aws-bedrock/admin-configuration)을 참고하세요.
</Info>

Cline이 AWS Bedrock을 사용하도록 구성하여 Amazon 관리형 서비스를 통해 Claude 및 기타 파운데이션 모델에 엔터프라이즈 액세스를 제공합니다.

## 구성 형식

원격 구성 JSON에서 `providerSettings.AwsBedrock` 섹션으로 Bedrock을 구성합니다:

```json
{
  "providerSettings": {
    "AwsBedrock": {
      "models": [
        {
          "id": "anthropic.claude-3-5-sonnet-20241022-v2:0",
          "name": "Claude 3.5 Sonnet"
        }
      ],
      "awsRegion": "us-east-1"
    }
  }
}
```

## 구성 필드

| 필드 | 타입 | 설명 | 필수 |
|-------|------|-------------|----------|
| `models` | Array | 모델 구성 목록 | 예 |
| `awsRegion` | String | AWS 리전(예: `us-east-1`) | 예 |
| `awsUseCrossRegionInference` | Boolean | 교차 리전 추론 활성화 | 아니오 |
| `awsUseGlobalInference` | Boolean | 글로벌 추론 라우팅 활성화 | 아니오 |
| `awsBedrockUsePromptCache` | Boolean | 프롬프트 캐싱 활성화 | 아니오 |
| `awsBedrockEndpoint` | String | 커스텀 Bedrock 엔드포인트 URL | 아니오 |
| `customModels` | Array | 커스텀 모델 구성 | 아니오 |

### 모델 구성

`models` 배열의 각 모델은 다음을 포함해야 합니다:

```json
{
  "id": "anthropic.claude-3-5-sonnet-20241022-v2:0",
  "name": "Claude 3.5 Sonnet",
  "info": {
    "maxTokens": 8192,
    "contextWindow": 200000,
    "supportsImages": true,
    "supportsPromptCache": true
  }
}
```

## 일반적인 모델 ID

| 모델 ID | 설명 | 컨텍스트 윈도우 |
|----------|-------------|----------------|
| `anthropic.claude-3-5-sonnet-20241022-v2:0` | 최신 Claude Sonnet | 200K 토큰 |
| `anthropic.claude-3-5-haiku-20241022-v1:0` | 최신 Claude Haiku | 200K 토큰 |
| `anthropic.claude-3-opus-20240229-v1:0` | Claude Opus | 200K 토큰 |

<Note>
모델 사용 가능 여부는 리전에 따라 다릅니다. 리전별 모델 가용성은 [AWS Bedrock 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/models-regions.html)를 참고하세요.
</Note>

## 구성 예시

### 기본 구성

```json
{
  "providerSettings": {
    "AwsBedrock": {
      "models": [
        {
          "id": "anthropic.claude-3-5-sonnet-20241022-v2:0",
          "name": "Claude 3.5 Sonnet"
        }
      ],
      "awsRegion": "us-east-1"
    }
  }
}
```

### 프롬프트 캐싱 사용


## 사전 준비

Cline에서 Bedrock을 사용하기 전에 다음이 필요합니다:

1. Bedrock 접근이 활성화된 **AWS 계정**
2. Bedrock API 호출을 위한 **IAM 권한**(`bedrock:InvokeModel`, `bedrock:InvokeModelWithResponseStream`)
3. Bedrock 콘솔에서 원하는 모델에 대한 **모델 접근 활성화**
4. **AWS 자격 증명** 구성(IAM 역할, 액세스 키, AWS 프로필)

<Tip>
AWS 계정 설정 및 IAM 구성은 [AWS Bedrock 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/getting-started.html)를 참고하세요.
</Tip>

## 문제 해결

**"Access Denied" 오류**

AWS 자격 증명에 필요한 Bedrock 권한이 있는지 확인하세요. 권한 요구사항은 [AWS IAM 문서](https://docs.aws.amazon.com/bedrock/latest/userguide/security-iam.html)를 참고하세요.

**"Model Not Found" 오류**

AWS Bedrock 콘솔에서 모델 접근이 활성화되어 있고 구성한 리전에서 모델이 उपलब्ध한지 확인하세요.

**지연 시간 증가**

사용자와 가까운 리전을 선택하거나 성능 개선을 위해 교차 리전 추론을 활성화하는 것을 고려하세요.
