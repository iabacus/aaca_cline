## 제공자 구성은 무엇인가요?

Cline의 제공자 구성을 통해 관리자는 다음을 할 수 있습니다:

1. **자격 증명 중앙 관리**: API 키와 인증 정보를 한 곳에서 저장
2. **모델 접근 제어**: 팀에서 사용할 수 있는 모델을 지정
3. **제공자 사용 강제**: 승인된 제공자만 사용하도록 제한

## 동작 방식

제공자 설정은 원격 구성 JSON 파일에서 구성합니다:

```json
{
  "providerSettings": {
    "provider": "bedrock",
    "bedrockRegion": "us-east-1",
    "bedrockServiceRole": "arn:aws:iam::..."
  }
}
```

구성되면:
- 모든 팀원에게 자동 적용
- 개별 사용자 설정을 덮어씀
- 팀 전체에서 일관된 제공자 사용 보장

## 구성 옵션

### 제공자 선택

지원 제공자:
- **bedrock**: AWS Bedrock 사용
- **vertex**: Google Vertex AI 사용  
- **openai**: OpenAI API 사용
- **azure**: Azure OpenAI 사용
- **litellm**: LiteLLM 프록시 사용

### 인증

제공자별 인증 방식이 다릅니다:

**AWS Bedrock:**
- 교차 계정 접근이 가능한 IAM 역할
- 액세스 키(프로덕션 권장 안 함)

**Google Vertex AI:**
- 서비스 계정 JSON 키
- Workload Identity(GKE 배포 시)

**OpenAI/Azure:**
- API 키

**LiteLLM:**
- 엔드포인트 URL + API 키

## 구성 예시

### IAM 역할을 사용하는 AWS Bedrock
```json
{
  "providerSettings": {
    "provider": "bedrock",
    "bedrockRegion": "us-east-1",
    "bedrockServiceRole": "arn:aws:iam::123456789012:role/ClineBedrockRole"
  }
}
```

### Google Vertex AI
```json
{
  "providerSettings": {
    "provider": "vertex",
    "vertexProject": "my-project-id",
    "vertexRegion": "us-central1"
  }
}
```

### LiteLLM 프록시
```json
{
  "providerSettings": {
    "provider": "litellm",
    "litellmBaseUrl": "https://litellm.company.com",
    "litellmApiKey": "sk-..."
  }
}
```
