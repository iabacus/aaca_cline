<Warning>
**자체 호스팅 구성 경로**

이 섹션은 복잡한 보안, 컴플라이언스, 다중 환경 요구사항이 있는 **자체 호스팅 Cline 인프라**를 배포하는 엔터프라이즈를 위한 내용입니다. 구성은 YAML 파일, Kubernetes/Helm 배포, Infrastructure-as-Code로 진행합니다.

**간단한 설정을 원하시나요?** 인프라 배포 없이 app.cline.bot 관리자 콘솔에서 웹 기반으로 빠르게 구성하려면 [SaaS 제공자 구성](/enterprise-solutions/configuration/remote-configuration/overview)을 참고하세요.
</Warning>

자체 호스팅 구성은 AI 제공자부터 커스텀 워크플로까지, 조직의 인프라 위에서 Cline 배포 전반을 중앙에서 제어할 수 있게 합니다. 이 섹션은 고급 보안, 컴플라이언스, 운영 기능을 포함해 엔터프라이즈 Cline 설치를 구성, 관리, 최적화하는 방법을 다룹니다.

## 시작하기

1. **평가**: 현재 인프라 및 통합 요구사항 검토
2. **제공자 설정**: 엔터프라이즈 자격 증명으로 선호 AI 제공자 구성
3. **보안 구성**: 규칙 및 접근 제어 구현
4. **모니터링 설정**: 운영 가시성을 위한 텔레메트리/모니터링 활성화
5. **사용자 온보딩**: 개발팀에 구성 배포

## 엔터프라이즈 아키텍처 고려사항

### 보안 및 컴플라이언스
- **제로 트러스트 아키텍처**: 모든 구성은 제로 트러스트 보안 모델을 지원
- **감사 로깅**: 모든 구성 변경에 대한 완전한 감사 추적
- **역할 기반 접근**: 관리 역할별 세밀한 권한
- **데이터 주권**: 민감 데이터를 인프라 경계 내에 유지

### 확장성 및 성능
- **다중 리전 지원**: 여러 지역에 구성 배포  
- **로드 밸런싱**: AI 제공자 요청을 여러 엔드포인트로 분산
- **캐싱 전략**: 지능형 캐싱으로 성능 최적화
- **레이트 리미팅**: 설정 가능한 속도 제한으로 남용 방지

### 통합 및 자동화
- **GitOps 통합**: 구성도 코드와 함께 버전 관리
- **CI/CD 파이프라인 통합**: 구성 배포 자동화
- **웹훅 지원**: 구성 변경에 맞춘 커스텀 자동화 트리거
- **API 우선 설계**: 모든 구성을 프로그래밍 방식으로 관리

## 구성 관리

모든 엔터프라이즈 구성은 다음을 지원합니다:

- **버전 관리**: 전체 변경 이력 추적
- **환경 승격**: dev → staging → production 순 배포
- **롤백 기능**: 문제 발생 시 신속한 복구
- **구성 검증**: 구성 변경에 대한 자동 테스트
- **드리프트 감지**: 구성 불일치 모니터링 및 알림
