Cline에는 사용 패턴을 이해하고 제품을 개선하기 위한 텔레메트리가 포함되어 있습니다. 사용자는 이 데이터 공유 여부를 직접 제어할 수 있습니다.

## Cline 텔레메트리란?

텔레메트리는 다음과 같은 익명 사용 이벤트를 수집합니다:

- 사용된 기능(도구, 명령, 워크플로)
- 작업 완료율
- 오류 발생
- 성능 지표

<Info>
모든 텔레메트리 데이터는 **익명**이며 코드 내용, 파일 내용, 기타 민감 정보를 포함하지 않습니다.
</Info>

## 사용자 제어

### Cline 텔레메트리 활성화/비활성화

개별 사용자는 Cline 설정에서 텔레메트리를 제어할 수 있습니다:

1. Cline 설정 열기
2. "Cline Telemetry" 토글 찾기
3. 원하는 대로 켜거나 끄기

변경 사항은 즉시 적용됩니다.

### 수집되는 항목

텔레메트리가 활성화되면 다음을 수집합니다:

<AccordionGroup>
  <Accordion title="기능 사용" icon="cursor-click">
    - 실행된 도구(예: read_file, execute_command)
    - 사용된 슬래시 명령
    - 트리거된 워크플로
    - 변경된 설정
  </Accordion>
  
  <Accordion title="작업 메트릭" icon="tasks">
    - 작업 시작/완료 이벤트
    - 모드 전환(Plan/Act)
    - 체크포인트 사용
    - 작업 소요 시간
  </Accordion>
  
  <Accordion title="오류 이벤트" icon="triangle-exclamation">
    - API 실패
    - 도구 실행 오류
    - 시스템 오류
    - 오류 유형 및 빈도
  </Accordion>
</AccordionGroup>

### 수집하지 않는 항목

Cline 텔레메트리는 **절대** 다음을 포함하지 않습니다:

- 코드 또는 파일 내용
- 파일 경로 또는 이름
- 명령 인자/파라미터
- 대화 내용
- 개인정보
- API 키 또는 자격 증명

## 엔터프라이즈 구성

관리자는 원격 구성으로 기본 텔레메트리 상태를 설정할 수 있습니다:

```json
{
  "telemetryEnabled": true
}
```

<Note>
엔터프라이즈 구성이 있어도 개별 사용자는 로컬 설정에서 Cline 텔레메트리를 끌 수 있습니다.
</Note>

## 고급 모니터링

더 상세한 모니터링이 필요한 조직은 OpenTelemetry 통합으로 텔레메트리 데이터를 자체 관측성 시스템으로 내보낼 수 있습니다.

사용 가능한 모니터링 옵션은 [엔터프라이즈 모니터링](/enterprise-solutions/monitoring/overview)을 참고하세요.

## 텔레메트리가 중요한 이유

익명 사용 데이터는 다음에 도움을 줍니다:

- **버그 식별**: 사용자에게 영향을 주는 문제 발견
- **기능 우선순위 결정**: 가장 많이 쓰는 기능에 집중
- **성능 개선**: 느린 동작을 찾아 개선
- **안정성 향상**: 오류율 추적 및 감소
