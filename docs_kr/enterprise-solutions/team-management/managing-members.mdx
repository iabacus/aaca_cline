효과적인 멤버 관리는 보안을 유지하고 팀의 생산성을 높이는 데 필수입니다. 이 가이드는 역할, 권한, 일상적인 멤버 관리에 필요한 모든 내용을 다룹니다.

## 역할 이해하기

보안과 생산성의 균형을 위해 각 팀원에게 적절한 역할을 부여하세요. 역할별 목적은 다음과 같습니다:

<CardGroup cols={3}>
  <Card title="Owner" icon="crown" color="#9D4EDD">
    **주요 계정 보유자**
    
    결제, 보안, 소유권 이전을 포함한 모든 설정에 무제한 접근. 1~2명의 핵심 리더에게만 부여하세요.
  </Card>
  
  <Card title="Admin" icon="user-gear" color="#7209B7">
    **팀 리드 및 IT 관리자**
    
    사용자 관리 및 제공자 구성 가능. 결제 권한 없이 운영 제어가 필요한 신뢰할 수 있는 관리자에게 적합합니다.
  </Card>
  
  <Card title="Member" icon="user" color="#560BAD">
    **개발자 및 기여자**
    
    공유 리소스로 Cline 사용 가능하지만 설정 변경 불가. 대부분의 팀원에게 가장 안전한 기본 역할입니다.
  </Card>
</CardGroup>

## 권한 매트릭스

역할별 권한을 정확히 이해하세요:

| 권한 | Member | Admin | Owner |
| :--- | :---: | :---: | :---: |
| **일반 사용** | | | |
| Cline 사용 | ✅ | ✅ | ✅ |
| 공유 API 제공자 접근 | ✅ | ✅ | ✅ |
| | | | |
| **멤버 관리** | | | |
| 멤버 조회 | ❌ | ✅ | ✅ |
| 새 멤버 초대 | ❌ | ✅ | ✅ |
| 멤버 역할 변경 | ❌ | ✅ | ✅ |
| 멤버 제거 | ❌ | ✅ | ✅ |
| Admin 제거 | ❌ | ❌ | ✅ |
| | | | |
| **구성** | | | |
| API 제공자 구성 | ❌ | ✅ | ✅ |
| 보안 설정 관리 | ❌ | ❌ | ✅ |
| | | | |
| **결제 및 소유권** | | | |
| 결제 정보 조회 | ❌ | ❌ | ✅ |
| 구독 관리 | ❌ | ❌ | ✅ |
| 소유권 이전 | ❌ | ❌ | ✅ |

<Note>
**빠른 참고:** 대부분의 사용자는 **Member**가 적합합니다. 사용자/구성을 관리해야 하는 경우에만 **Admin**을 부여하세요. **Owner**는 1~2명의 계정 리더로 제한하세요.
</Note>

## 멤버 관리 작업

<Tabs>
  <Tab title="멤버 추가">
    ### 새 팀원 초대
    
    1. **Members로 이동**
       - app.cline.bot의 조직 대시보드로 이동
       - 사이드바의 "Members" 클릭
    
    2. **초대 전송**
       - "Invite Member" 클릭
       - 사용자 이메일 입력(검증된 도메인이어야 함)
       - 역할 선택(Member, Admin, Owner)
       - "Send Invite" 클릭
    
    3. **초대 상태 확인**
       - 초대된 사용자는 이메일로 참여 링크를 받음
       - 대기 중인 초대는 멤버 목록에 "Pending"으로 표시
       - 대기 초대 1건당 라이선스 좌석 1개가 차지됨
    
    <Tip>
    **대량 초대:** 여러 사용자를 한 번에 추가해야 한다면 support@cline.bot에 문의해 CSV 일괄 초대를 지원받을 수 있습니다.
    </Tip>
  </Tab>
  
  <Tab title="역할 변경">
    ### 멤버 권한 변경
    
    1. **멤버 찾기**
       - Members 페이지로 이동
       - 변경할 사용자를 찾기
    
    2. **역할 변경**
       - 현재 역할 옆의 드롭다운 클릭
       - 새 역할 선택
       - 변경 확인
    
    3. **즉시 적용**
       - 역할 변경은 즉시 적용됨
       - 사용자가 권한 변화를 보려면 로그아웃 후 재로그인이 필요할 수 있음
    
    <Warning>
    **Admin → Member:** Admin을 Member로 내리면 사용자/구성 관리 권한이 즉시 회수됩니다. 해당 권한이 더 이상 필요하지 않은지 확인하세요.
    </Warning>
  </Tab>
  
  <Tab title="멤버 제거">
    ### 팀원 오프보딩
    
    1. **멤버 목록 접근**
       - 조직의 Members 페이지로 이동
       - 제거할 사용자를 찾기
    
    2. **사용자 제거**
       - 이름 옆의 메뉴 아이콘(⋮) 클릭
       - "Remove from Organization" 선택
       - 제거 확인
    
    3. **즉시 효과**
       - 사용자는 즉시 조직 접근 권한 상실
       - 좌석이 해제되어 다른 사용자에게 할당 가능
       - 감사 로그는 컴플라이언스 용도로 보존
    
    <Info>
    **데이터 보존:** 멤버 제거는 과거 활동 로그를 삭제하지 않습니다. 모든 감사 추적은 컴플라이언스 목적으로 유지됩니다.
    </Info>
  </Tab>
  
  <Tab title="초대 취소">
    ### 대기 초대 취소
    
    초대된 사용자가 아직 수락하지 않았다면 초대를 취소할 수 있습니다:
    
    1. Members 목록에서 대기 초대 찾기
    2. "Revoke Invitation" 클릭
    3. 좌석이 즉시 해제되어 다른 사용자에게 할당 가능
    
    다음과 같은 경우 유용합니다:
    - 잘못된 이메일로 초대했을 때
    - 사용자가 더 이상 접근이 필요 없을 때
    - 좌석을 긴급 재할당해야 할 때
  </Tab>
</Tabs>

## ID 및 접근 요구사항

사용자가 조직에 참여하려면 다음 두 조건을 만족해야 합니다:

<Steps>
  <Step title="검증된 ID 제공자">
    조직은 다음과 같은 검증된 **ID 제공자(IdP)**를 사용해야 합니다:
    - Microsoft Entra ID(Azure AD)
    - Okta
    - Google Workspace
    - AWS IAM Identity Center
    
    사용자는 IdP를 통해 인증해야 조직에 접근할 수 있습니다.
  </Step>
  
  <Step title="도메인 검증">
    조직은 **검증된 도메인**을 보유해야 합니다. 도메인 제공자(예: Google, Microsoft, Cloudflare)를 통해 소유권을 검증해야 합니다.
    
    검증된 도메인의 이메일 주소를 가진 사용자만 참여할 수 있습니다.
  </Step>
</Steps>

<Note>
이 요구사항은 회사 소속의 인증된 사용자만 Cline 조직에 접근하도록 보장하여 무단 접근을 방지합니다.
</Note>

## 좌석 관리

좌석이 어떻게 계산되는지 이해하면 라이선스를 효율적으로 관리할 수 있습니다:

<AccordionGroup>
  <Accordion title="좌석 계산 방식" icon="chair">
    - 각 사용자(Owner/Admin/Member)는 **좌석 1개**를 사용
    - 대기 중인 초대도 좌석 1개를 차지
    - 멤버 제거 또는 초대 취소 시 좌석 즉시 해제
    - 라이선스가 사용 가능한 최대 좌석 수를 결정
  </Accordion>
  
  <Accordion title="좌석이 사용되는 시점" icon="user-plus">
    좌석이 사용되는 경우:
    - 초대장을 보내면("pending" 상태)
    - 초대된 사용자가 수락하여 참여
    - SSO를 통해 기존 사용자가 접근 권한을 부여받음
  </Accordion>
  
  <Accordion title="좌석 해제" icon="user-minus">
    좌석을 해제하려면:
    - 활성 멤버 제거
    - 대기 초대 취소
    - (설정된 경우) 초대 만료 대기
  </Accordion>
  
  <Accordion title="라이선스 업그레이드" icon="arrow-up">
    좌석이 더 필요하신가요?
    - **Teams 플랜:** 계정 담당자에게 문의하거나 app.cline.bot/settings/billing에서 업그레이드
    - **Enterprise 플랜:** 사용자 수 제한 없는 무제한 좌석 포함
  </Accordion>
</AccordionGroup>

다음 지침으로 조직 보안을 강화하세요:

<CardGroup cols={2}>
  <Card title="최소 권한 원칙" icon="shield-check">
    필요한 최소 역할만 부여하세요. 대부분의 사용자는 Member가 적합합니다. 업무상 필요할 때만 Admin/Owner를 부여하세요.
  </Card>
  
  <Card title="Owner 역할 제한" icon="user-lock">
    결제와 보안을 관리하는 1~2명의 핵심 인물만 Owner로 지정하세요. 중요한 설정의 실수 또는 악의적 변경을 방지합니다.
  </Card>
  
  <Card title="정기 감사" icon="clipboard-check">
    분기마다 멤버 목록을 검토하세요. 비활성 사용자는 즉시 제거하고 Admin/Owner 역할이 적절한지 확인하세요.
  </Card>
  
  <Card title="오프보딩 프로세스" icon="door-open">
    표준 오프보딩 체크리스트를 마련하세요: Cline에서 제거, IdP 접근 회수, 감사 로그 기록, 주요 책임 재할당.
  </Card>
</CardGroup>

<Warning>
**Owner 책임:** Owner는 결제와 소유권 이전 권한이 있으므로 신중히 선택하고 조직의 보안 정책에 이를 문서화하세요.
</Warning>

## 고급 시나리오

<AccordionGroup>
  <Accordion title="소유권 이전" icon="exchange">
    소유권 이전은 현재 Owner만 수행할 수 있습니다:
    
    1. 조직 설정으로 이동
    2. "Ownership" 섹션으로 이동
    3. 멤버 목록에서 새 Owner 선택
    4. 인증 후 이전 확인
    5. 새 Owner가 즉시 제어권을 획득
    
    **중요:** 이 작업은 이전 Owner가 되돌릴 수 없습니다. 필요 시 새 Owner가 역이전을 진행해야 합니다.
  </Accordion>
  
  <Accordion title="여러 Admin 관리" icon="users-gear">
    여러 Admin이 있는 경우:
    
    - 각 Admin의 책임 영역을 문서화
    - 감사 로그로 구성 변경 추적
    - 대규모 팀은 로테이션 스케줄 고려
    - Owner 수준 의사결정에 대한 에스컬레이션 경로 정의
  </Accordion>
  
  <Accordion title="임시 접근" icon="clock">
    계약직/임시 인력의 경우:
    
    - 멤버로 생성하고 만료 캘린더 리마인더 설정
    - 내부 시스템에 접근 기간 문서화
    - 계약 종료 시 제거하도록 캘린더 리마인더
    - IdP가 지원한다면 기간 제한 계정 사용 고려
  </Accordion>
</AccordionGroup>

## 문제 해결

<AccordionGroup>
  <Accordion title="사용자가 초대를 수락할 수 없음" icon="circle-exclamation">
    **일반 원인:**
    - 이메일 도메인이 검증된 도메인과 일치하지 않음
    - 사용자의 IdP 접근 권한 미부여
    - 초대 링크 만료
    
    **해결:** 도메인 검증이 완료되었는지 확인하고 초대를 재전송하세요.
  </Accordion>
  
  <Accordion title="Admin 제거 불가" icon="user-slash">
    **원인:** Admin은 Owner만 제거할 수 있습니다.
    
    **해결:** Owner에게 제거를 요청하세요. 조직의 유일한 Owner를 제거해야 한다면 support@cline.bot에 문의하세요.
  </Accordion>
  
  <Accordion title="좌석 부족" icon="triangle-exclamation">
    **라이선스 한도를 초과한 경우:**
    - 비활성 멤버 제거로 좌석 확보
    - 불필요한 대기 초대 취소
    - 라이선스 업그레이드
  </Accordion>
</AccordionGroup>
