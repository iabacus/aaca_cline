## 개요
Cline Enterprise는 **WorkOS AuthKit**을 통해 ID 제공자(IdP)와 SSO를 통합합니다.

이 페이지는 WorkOS AuthKit을 사용하는 Cline Enterprise SSO 설정을 개략적으로 설명합니다.

초기 온보딩을 완료하지 않았다면 [온보딩](/enterprise-solutions/onboarding)부터 시작하세요.

## 설정 위치
SSO 설정은 두 곳에서 진행됩니다:

1) **Cline 대시보드(app.cline.bot)**
- 조직의 SSO가 정상 동작하는지 로그인/검증하는 곳

2) **WorkOS 대시보드**
- IdP 연결을 구성하는 곳(AuthKit → Connections). 엔터프라이즈 온보딩 중 지정된 관리자가 접근 권한을 받습니다.

## Cline 대시보드 사용
https://app.cline.bot 에서 다음을 수행합니다:
- 로그인 및 온보딩 플로우 완료
- SSO로 사용자 인증이 가능한지 검증

## WorkOS에서 IdP 연결 구성

엔터프라이즈 온보딩 중 지정된 관리자가 WorkOS에서 조직의 대시보드에 접근할 수 있는 초대 이메일을 받습니다.

<Frame>
  <img src="/assets/workos-invite-email.png" alt="WorkOS 초대 이메일 예시" />
</Frame>

Cline Enterprise에 IdP를 연결하려면 **WorkOS AuthKit**에서 IdP를 구성합니다:

1. WorkOS 대시보드에서 **AuthKit → Connections**로 이동
2. **Add Connection** 클릭
3. ID 제공자 선택(예: Okta, Microsoft Entra ID/Azure AD, Google Workspace, Generic SAML/OIDC)
4. WorkOS의 제공자별 안내에 따라 구성

WorkOS의 UI와 필수 필드는 제공자별로 다릅니다. 자세한 내용은 WorkOS 문서를 참고하세요:
- https://workos.com/docs/authkit/sso

## Keycloak 참고(IdP 호환성)
Cline Enterprise의 기본 SSO 통합은 **WorkOS**를 통해 제공됩니다.

IdP로 **Keycloak**을 사용하는 경우, WorkOS에서 **Generic SAML** 또는 **Generic OIDC** 제공자로 Keycloak을 구성하는 방식이 지원됩니다(WorkOS에서 요청하는 해당 설정 사용).

## 검증
WorkOS 구성을 완료한 후:

1) https://app.cline.bot 에서 SSO 로그인을 시도합니다.
2) 로그인 완료 후 정상적으로 다시 리다이렉트되는지 확인합니다.

## 문제 해결
- **Redirect URI 불일치**: WorkOS에 구성된 리다이렉트/콜백 URL이 Cline Enterprise 온보딩 때 제공된 값과 일치하는지 확인하세요.

추가 문제 해결은 WorkOS 문서를 참고하세요:
- https://workos.com/docs/authkit/sso
