폴더 멘션을 사용하면 전체 디렉터리를 Cline 대화에 가져올 수 있습니다. `@/` 다음에 슬래시로 끝나는 폴더 경로를 입력하면, Cline이 폴더 구조와 내용을 볼 수 있습니다.

채팅에서 `@/`를 입력하면 Cline이 워크스페이스의 파일과 폴더를 보여줍니다. 원하는 폴더로 이동한 뒤 끝에 슬래시를 붙이면, Cline이 해당 폴더의 구조와 내용을 확인합니다.

저는 코드베이스의 특정 영역 전체를 이해하거나 리팩터링할 때 폴더 멘션을 사용합니다. 파일을 하나씩 참조하는 대신 디렉터리를 그대로 지정할 수 있습니다:

```
I'm trying to understand how the authentication flow works in my app.
Can you explain the structure and relationships between the files in @/src/auth/?
```

이렇게 하면 Cline은 auth 디렉터리의 모든 파일과 내용을 보고 서로 어떤 관계가 있는지 설명할 수 있습니다. 여러 파일이 얽힌 복잡한 흐름을 이해하는 데 필요한 전체 컨텍스트를 제공합니다.

폴더 멘션은 프로젝트 구조를 점검할 때도 유용합니다. 구조가 적절한지 고민될 때 Cline에게 리뷰를 요청할 수 있습니다:

```
I'm setting up a new React project. Does this folder structure make sense? @/src/
What would you change to make it more maintainable as the project grows?
```

여러 관련 파일을 다뤄야 할 때는 개별 파일을 참조하는 대신 폴더 멘션을 사용해 보세요. Cline이 전체 구조를 보며 더 포괄적인 도움을 줄 수 있습니다.

## 내부 동작 방식

폴더 멘션을 사용할 때 내부적으로는 다음과 같이 동작합니다:

1. 메시지를 전송하면 Cline이 텍스트에서 `@/path/to/folder/` 패턴(끝에 슬래시 포함)을 감지
2. 확장이 워크스페이스 루트 기준으로 폴더 경로를 해석
3. `fs.readdir()`로 폴더 내 파일/하위 디렉터리 목록을 가져옴
4. 각 파일이 바이너리인지 텍스트인지 확인
5. 텍스트 파일은 전체 내용을 추출
6. 폴더 구조와 파일 내용이 구조화된 형식으로 메시지에 첨부됨:

    ```
    <folder_content path="path/to/folder">
    ├── file1.txt
    ├── file2.js
    └── subfolder/

    <file_content path="path/to/folder/file1.txt">
    [File content]
    </file_content>

    <file_content path="path/to/folder/file2.js">
    [File content]
    </file_content>
    </folder_content>
    ```

7. 폴더 구조와 파일 내용이 포함된 메시지가 AI로 전송
8. AI는 디렉터리 구조와 파일 내용을 모두 “볼” 수 있음

이 과정은 폴더 멘션을 사용할 때마다 자동으로 수행되어, AI가 프로젝트 구조와 파일 내용을 종합적으로 이해할 수 있게 해줍니다.
