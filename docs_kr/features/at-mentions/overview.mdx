@ 멘션은 Cline의 가장 강력한 기능 중 하나로, 외부 컨텍스트를 대화에 자연스럽게 가져올 수 있습니다. 코드, 에러 메시지, 문서를 복사/붙여넣기 하는 대신, @ 기호로 참조만 하면 됩니다.

</Frame>

채팅 입력창에서 `@`를 입력하면, 사용할 수 있는 멘션 유형 목록이 표시됩니다. 파일, 폴더, 문제, 터미널 출력, git 변경, 웹 콘텐츠 등을 대화에 바로 포함할 수 있습니다.

## @ 멘션이 중요한 이유

@ 멘션은 다음과 같은 방식으로 Cline과의 상호작용을 바꿉니다:

1. **복사/붙여넣기 제거**: 코드, 오류 메시지, 터미널 출력 등을 직접 복사하지 않아도 됩니다.

2. **컨텍스트 유지**: import, 관련 함수, 주변 코드까지 포함한 전체 컨텍스트를 제공합니다.

3. **포맷 유지**: 터미널 출력, 오류 메시지, 웹 콘텐츠의 포맷이 그대로 유지됩니다.

4. **복합 워크플로 지원**: 여러 @ 멘션을 조합해 문제의 전체 맥락을 제공합니다:

    ```
    I'm getting these errors: @problems

    Here's my component: @/src/components/Form.jsx
    And the API endpoint: @/src/api/users.js

    The error happens when I submit: @terminal

    I think this commit might have caused it: @a1b2c3d
    ```

## 시작하기

@ 멘션을 사용하려면:

1. 채팅 입력창에 `@` 입력
2. 메뉴에서 멘션 유형 선택 또는 계속 입력
3. 파일/폴더는 워크스페이스 구조를 탐색
4. 평소처럼 메시지 전송

Cline이 멘션을 자동 처리해 참조된 내용을 AI 컨텍스트에 포함합니다.

다음 대화에서 @ 멘션을 사용해 보세요. 외부 컨텍스트를 자연스럽게 포함할 수 있어 훨씬 효율적인 대화를 할 수 있습니다.

## 내부 동작 방식

@ 멘션을 사용할 때 내부적으로는 다음 과정이 진행됩니다:

1. **감지**: 메시지 전송 시 정규식으로 @ 멘션 패턴을 스캔
2. **처리**: 각 멘션 유형별로 처리
    - 멘션 유형 결정(파일, 폴더, 문제, 터미널, git, URL)
    - 관련 콘텐츠 가져오기(파일 내용, 터미널 출력 등)
    - 적절한 형식으로 포맷
3. **메시지 확장**: 원본 메시지에 구조화된 데이터를 첨부

    ```
    Your original message with @/path/to/file

    <file_content path="/path/to/file">
    [Complete file content]
    </file_content>
    ```

4. **컨텍스트 포함**: 확장된 메시지가 AI 모델에 전달됨
5. **응답**: AI가 복사/붙여넣기처럼 모든 참조 콘텐츠를 확인할 수 있음

이 과정은 @ 멘션을 사용할 때마다 자동으로 수행되어, 사용자가 직접 복사하지 않아도 AI에게 완전한 컨텍스트를 제공합니다.

각 @ 멘션 유형의 자세한 구현은 해당 문서 페이지에서 확인할 수 있습니다.
