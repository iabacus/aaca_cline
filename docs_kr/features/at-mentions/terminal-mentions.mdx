터미널 멘션을 사용하면 터미널 출력을 Cline 대화에 바로 가져올 수 있습니다. `@terminal`을 입력하면 최근 터미널 출력이 Cline에 전달됩니다.

채팅에서 `@`를 입력하고 메뉴에서 "Terminal"을 선택하거나 `@terminal`을 직접 입력하세요. Cline은 활성 터미널의 최근 출력(에러 메시지, 빌드 로그, 명령 결과 등)을 즉시 확인합니다.

저는 빌드 오류, 테스트 실패, 디버깅 출력 문제를 다룰 때 터미널 멘션을 자주 사용합니다. 터미널 출력은 복사/붙여넣기 할 때 포맷이 깨지는 경우가 많으므로, 이렇게 요청합니다:

```
I'm getting this error when running my tests: @terminal

What's causing this and how can I fix it?
```

이렇게 하면 Cline이 포맷이 유지된 전체 터미널 출력을 확인할 수 있습니다. 오류 메시지, 스택 트레이스, 주변 컨텍스트를 분석해 더 정확한 도움을 제공할 수 있습니다.

터미널 멘션은 파일 멘션과 함께 사용할 때 특히 유용합니다. 실패한 API 호출을 디버깅할 때는 다음과 같이 참조할 수 있습니다:

```
I'm getting this error when calling my API: @terminal

Here's my API client code: @/src/api/client.js
And the endpoint implementation: @/src/server/routes/users.js

What am I doing wrong?
```

이 방식은 정확한 오류 출력, 클라이언트 코드, 서버 구현까지 모두 제공하므로, 별도로 복사하지 않아도 Cline이 필요한 정보를 모두 볼 수 있습니다.

다음에 명령 출력이나 빌드 오류로 막히면 `@terminal`을 사용해 보세요. 터미널 컨텍스트 전체를 포맷 그대로 전달하므로 더 정확한 도움을 받을 수 있습니다.

## 내부 동작 방식

터미널 멘션을 사용할 때 내부적으로는 다음과 같이 동작합니다:

1. 메시지를 전송하면 Cline이 텍스트에서 `@terminal` 패턴을 감지
2. 확장이 `getLatestTerminalOutput()`을 호출하여 VS Code 터미널 API에 접근
3. 활성 터미널의 최근 출력 버퍼를 캡처
4. 터미널 출력이 구조화된 형식으로 메시지에 첨부됨:

    ```
    <terminal_output>
    $ npm run test
    > project@1.0.0 test
    > jest

    FAIL src/components/__tests__/Button.test.js
    ● Button component › renders correctly

    [Complete terminal output with formatting preserved]
    </terminal_output>
    ```

5. 터미널 출력이 포함된 메시지가 AI로 전송
6. AI가 포맷이 유지된 전체 터미널 출력을 확인 가능

이 과정은 터미널 멘션을 사용할 때마다 자동으로 수행되어, 명령 결과나 오류 출력을 수동으로 복사하지 않아도 AI가 전체 출력 맥락을 볼 수 있게 해줍니다.

## 터미널 문제 해결

터미널 멘션 또는 터미널 통합 전반에서 문제가 발생하는 경우(예: "Shell Integration Unavailable" 또는 출력 미표시), [터미널 통합 문제 해결 가이드](/troubleshooting/terminal-integration-guide)를 참고하세요.

일반적인 이슈:

-   터미널 멘션이 출력 내용을 캡처하지 못함
-   Cline 채팅에 "Shell Integration Unavailable" 메시지 표시
-   명령이 실행되지만 출력이 Cline에 표시되지 않음
-   터미널 통합이 간헐적으로 동작

해당 가이드는 플랫폼별 해결 방법과 자세한 설정 단계를 제공합니다.
