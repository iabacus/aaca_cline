파일 멘션을 사용하면 워크스페이스의 어떤 파일이든 대화에 바로 가져올 수 있습니다. 코드 조각을 복사/붙여넣기 할 필요 없이 `@/`를 입력하고 필요한 파일을 지정하세요.

채팅에서 `@/`를 입력하면 Cline이 워크스페이스 파일을 보여줍니다. 폴더를 탐색해 원하는 파일을 선택하면 즉시 Cline에서 사용할 수 있습니다. 관련 import, 함수, 주변 컨텍스트까지 함께 포함됩니다.

저는 디버깅할 때 파일 멘션을 자주 사용합니다. 어떤 코드를 복사해야 할지 고민하는 대신 파일을 직접 참조합니다:

```
I'm getting this error when my form submits: @terminal

Here's my component: @/src/components/ContactForm.jsx

And the API endpoint: @/src/api/contact.js

What am I missing?
```

이렇게 하면 Cline이 오류 메시지, 컴포넌트 코드, API 엔드포인트를 모두 확인할 수 있습니다. 제가 따로 복사하지 않아도 imports, 의존성, 주변 컨텍스트를 포함한 전체 상황을 파악할 수 있죠.

여러 파일에 걸친 복잡한 버그를 다룰 때 파일 멘션은 특히 유용합니다. 예전에는 관련 파일을 하나씩 복사해 놓치지 않도록 주의해야 했지만, 이제는 `@/`로 각 파일을 참조하기만 하면 됩니다.

다음에 막히는 문제가 생기면 코드 복사 대신 파일 멘션을 써보세요. 더 빠르게, 더 좋은 답을 얻을 수 있습니다.

## 내부 동작 방식

파일 멘션을 사용할 때 내부적으로는 다음과 같이 동작합니다:

1. 메시지를 전송하면 Cline이 텍스트에서 `@/path/to/file` 패턴을 감지
2. 확장이 워크스페이스 루트 기준으로 파일 경로를 해석
3. 파일이 바이너리인지(이미지 등) 텍스트인지 확인
4. 텍스트 파일이면 전체 내용을 읽음
5. 파일 내용을 구조화된 형식으로 메시지에 첨부:
    ```
    <file_content path="path/to/file">
    [Complete file content]
    </file_content>
    ```
6. 파일 내용이 포함된 메시지가 AI로 전송
7. AI는 복붙된 것처럼 파일 내용을 완전히 “볼” 수 있음

이 과정은 파일 멘션을 사용할 때마다 자동으로 수행되어, 사용자가 직접 복사할 필요 없이 AI에게 완전한 컨텍스트를 제공합니다.
