백그라운드 편집은 Cline이 diff 편집기를 열지 않고 파일을 변경하도록 해주어, Cline이 다른 파일을 처리하는 동안에도 계속 코딩할 수 있습니다.

<Note>
  이 기능은 실험적 기능으로 표시되어 있습니다.
</Note>

## 동작 방식

기본적으로 Cline은 수정하는 파일마다 나란히 보는 diff 편집기 탭을 엽니다. 백그라운드 편집을 활성화하면:

- 편집 내용이 새 탭 없이 파일에 바로 기록됩니다
- 변경 사항이 채팅 패널에 접을 수 있는 diff 블록으로 표시됩니다
- 에디터 포커스가 현재 열어둔 파일에 유지됩니다

## 백그라운드 편집 활성화

1. Cline 패널 오른쪽 상단의 설정 아이콘(톱니바퀴)을 클릭
2. "**Feature Settings**"로 이동
3. "**Enable Background Edit**" 토글을 켭니다

## 변경 사항 보기

파일 변경은 채팅 패널에 아래와 같이 표시됩니다:

- **파일 동작 아이콘**: 파일 추가/수정/삭제 여부 표시
- **통계**: 추가(+)와 삭제(-) 수를 한눈에 확인
- **접을 수 있는 diff**: 파일 헤더를 클릭해 펼치거나 접기
- **실시간 스트리밍**: 변경 사항이 줄 단위로 표시

초록색은 추가, 빨간색은 삭제를 의미합니다.

## 이런 경우에 적합

다음과 같은 상황에서 잘 맞습니다:

- [자동 승인 모드](/features/auto-approve)를 사용하고 변경 사항을 사후에 검토하고 싶을 때
- 작은 파일 변경이 많은 작업을 할 때
- 현재 작업 중인 파일에 집중하고 싶을 때

변경 사항을 저장하기 전에 하나씩 검토하거나, Cline의 제안을 즉시 편집해야 한다면 기본 diff 편집기를 유지하는 것이 좋습니다.

## 다른 기능과의 관계

- **체크포인트**: 파일 작업 후에도 계속 생성됩니다
- **자동 승인**: 끊김 없는 워크플로에 잘 맞습니다
- **메시지 편집**: 이전 메시지로 복원하는 동작이 동일하게 동작합니다
