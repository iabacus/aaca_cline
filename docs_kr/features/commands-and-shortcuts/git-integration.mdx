Cline의 Git 통합은 버전 관리 워크플로에 AI 도움을 직접 제공합니다. 에디터를 떠나지 않고 커밋 메시지를 생성할 수 있습니다.

## 커밋 메시지 생성

가장 유용한 Git 통합 기능 중 하나는 의미 있는 커밋 메시지를 자동으로 생성하는 기능입니다:

<Frame>
	<img
		src="https://storage.googleapis.com/cline_public_images/docs/assets/generate-commit-message-with-cline.png"
		alt="Cline으로 커밋 메시지 생성"
	/>
</Frame>

1. 변경 사항을 만들고 Git에 스테이징합니다
2. 소스 컨트롤 뷰에서 로봇 아이콘을 클릭하거나 "Generate Commit Message with Cline" 명령을 실행합니다
3. Cline이 변경 사항을 분석하고 설명적인 커밋 메시지를 생성합니다
4. 메시지가 커밋 메시지 입력 상자에 자동으로 삽입됩니다

생성된 커밋 메시지는:

-   간결한 요약(50~72자)으로 시작
-   명령형(예: "Add feature" 대신 "Added feature")을 사용
-   무엇이, 왜 변경되었는지 설명
-   Git 모범 사례를 따름

이 기능은 시간을 절약하고 커밋 히스토리를 일관되고 유익하게 유지하는 데 도움이 됩니다.

<Tip>
	채팅 메시지에서 `@git-changes` 및 `@[commit-hash]` 멘션을 사용하는 방법은 [Git 멘션](/features/at-mentions/git-mentions) 문서를 참고하세요.
</Tip>

## 동작 원리

Cline의 커밋 메시지 생성 기능을 사용하면 내부적으로 다음이 수행됩니다:

1. Cline이 `getWorkingState()`를 사용해 현재 Git diff를 가져옵니다
2. 이 diff를 AI용 특수 프롬프트로 포맷합니다
3. AI가 변경 사항을 분석해 적절한 커밋 메시지를 생성합니다
4. 메시지를 추출해 Git 커밋 메시지 입력 상자에 삽입합니다

이 과정은 현재 Cline API 구성에 따라 동작하므로, 선택한 AI 모델에 맞는 품질로 메시지가 생성됩니다.

## 효과적으로 사용하는 팁

-   **복잡한 변경에 사용**: AI는 여러 변경을 하나의 일관된 메시지로 요약하는 데 특히 강합니다.

-   **생성된 메시지 검토 및 수정**: AI가 고품질 메시지를 생성하더라도, 필요 시 검토하고 조정하는 것이 좋습니다.

-   **관련 변경을 함께 스테이징**: 최상의 결과를 위해 관련 변경을 함께 스테이징하면 더 일관된 메시지가 생성됩니다.

-   **일관된 커밋 히스토리 유지**: 커밋 메시지 생성 기능을 사용하면 히스토리 스타일을 통일할 수 있습니다.

## 내부 동작 방식

커밋 메시지 생성은 VS Code의 Git 확장 API를 활용해 저장소 정보를 가져옵니다:

1. 명령을 실행하면:
    - Cline이 현재 diff를 가져옵니다
    - 커밋 메시지 포맷 지침과 함께 AI에 전송합니다
    - AI 응답을 파싱합니다
    - Git 확장 API를 통해 커밋 메시지를 설정합니다

Git과의 통합 덕분에 워크플로를 깨뜨리지 않고도 고품질 커밋 메시지를 쉽게 생성할 수 있습니다.

다음에 좋은 커밋 메시지를 쓰기 어려울 때, Cline의 커밋 메시지 생성 기능을 사용해 보세요. 시간을 절약하고 필요한 자리에서 바로 AI 도움을 받아 버전 관리 워크플로를 개선할 수 있습니다.
