Cline의 터미널 통합은 터미널 출력을 Cline과의 대화에 바로 가져올 수 있게 해줍니다. 오류 메시지나 명령 결과를 복사/붙여넣기 할 필요 없이, 터미널에서 우클릭만으로 Cline에 보낼 수 있습니다.

## 우클릭 터미널 통합

VS Code 터미널에서 Cline과 논의하고 싶은 출력이 보이면:

1. 터미널에서 우클릭
2. 컨텍스트 메뉴에서 "Add to Cline" 선택
3. 터미널 출력이 즉시 Cline 채팅 패널로 전송됨

다음과 같은 상황에 적합합니다:

-   빌드 오류 디버깅
-   테스트 실패 이해
-   명령 출력 분석
-   오류 메시지 해결 도움

터미널 작업 중 문제를 만났을 때 특히 유용합니다.

Cline 채팅 패널로 전환해 문제를 설명하는 대신, 몇 번의 클릭만으로 터미널 출력을 바로 보낼 수 있습니다.

또는 [`@terminal`](/features/at-mentions/terminal-mentions) 멘션을 사용해 전체 터미널 출력을 Cline에 보낼 수도 있습니다.

<Tip>
	채팅 메시지에서 `@terminal` 멘션을 사용하는 방법은 [터미널 멘션](/features/at-mentions/terminal-mentions) 문서를 참고하세요.
</Tip>

## 터미널 통합 동작 방식

우클릭 터미널 통합을 사용하면 Cline은:

1. 모든 서식을 유지한 채 터미널 출력을 캡처
2. 명령 히스토리와 결과를 포함한 전체 컨텍스트를 포함
3. AI가 이해하기 쉽게 포맷팅
4. 사용자가 보는 그대로를 AI가 볼 수 있게 함

덕분에 Cline이 터미널 관련 이슈를 정확히 이해하고 도움을 줄 수 있습니다.

## 내부 동작 방식

터미널 통합은 출력 캡처를 위해 다음과 같은 기법을 사용합니다:

1. 통합을 실행하면 Cline이:

    - 현재 클립보드 내용을 임시 저장
    - 터미널 전체 내용을 선택(또는 사용자가 선택한 내용)
    - 클립보드에 복사
    - 클립보드를 읽어 터미널 내용을 가져옴
    - 원래 클립보드 내용을 복원

2. 그다음 터미널 내용은:
    - 올바른 구문 하이라이팅으로 포맷
    - 메시지에 추가되거나 새 메시지로 전송
    - 필요 시 추가 컨텍스트가 보강됨

이 방식은 색상과 서식을 포함한 모든 터미널 출력을 정확히 캡처하면서도 클립보드를 건드리지 않도록 보장합니다.

## 효과적으로 사용하는 팁

-   **오류 메시지에 터미널 통합 사용**: 터미널 오류를 Cline에 보내면 직접 설명하는 것보다 더 빠르게 해결되는 경우가 많습니다.

-   **필요 시 특정 출력만 선택**: 기본적으로 전체 내용을 캡처하지만, 우클릭 전에 특정 줄을 선택해 관련 출력만 보낼 수 있습니다.

-   **파일 멘션과 결합**: 터미널 출력을 보낸 뒤 @ 멘션으로 관련 파일을 함께 전달하면 질문의 맥락을 강화할 수 있습니다.

-   **빌드/테스트 출력에 컨텍스트 추가**: 여러 줄에 걸친 복잡한 빌드 오류나 테스트 실패를 이해하는 데 특히 유용합니다.

다음에 터미널의 난해한 오류 메시지를 볼 때, 복사/붙여넣기 대신 Cline의 터미널 통합을 사용해 보세요. Cline이 서식이 포함된 전체 터미널 컨텍스트를 볼 수 있어 더 정확한 도움을 받을 수 있습니다.

## 터미널 문제 해결

"Shell Integration Unavailable" 같은 문제나 출력이 보이지 않는 현상이 있다면 [터미널 통합 문제 해결 가이드](/troubleshooting/terminal-integration-guide)를 참고하세요.

문제 해결 가이드에서는 다음 내용을 다룹니다:

-   일반적인 터미널 통합 문제와 빠른 해결책
-   Windows, macOS, Linux 플랫폼별 해결책
-   zsh, bash, PowerShell 등 셸별 구성
-   고급 디버깅 기법
-   터미널 설정 최적화

<Tip>
	**빠른 해결:** 대부분의 터미널 문제는 Cline 설정에서 셸을 bash로 변경하고 셸 통합 타임아웃을 10초로 늘리면 해결됩니다.
</Tip>
