체크포인트는 작업의 각 단계가 끝날 때 워크스페이스 스냅샷을 자동으로 저장합니다. 변경 사항을 추적하고 필요 시 롤백하며, 안심하고 코드를 실험할 수 있게 해줍니다.

## 체크포인트 동작 방식

Cline은 각 도구 사용(파일 편집, 명령 실행 등) 후 체크포인트를 생성합니다. 이 체크포인트는:

- Git 워크플로와 충돌 없이 나란히 동작
- 복원 사이의 컨텍스트를 유지
- 섀도우 Git 저장소로 변경 사항을 추적

예를 들어 기능 개발 중 Cline이 여러 파일을 변경하면 각 변경마다 체크포인트가 생성됩니다. 덕분에 각 수정 사항을 검토하고, 필요하면 메인 Git 저장소에 영향을 주지 않고 어느 시점으로든 되돌릴 수 있습니다.

## 체크포인트 활성화/비활성화

체크포인트는 기본적으로 활성화되어 있습니다. 토글하려면:

1. Cline 패널의 톱니바퀴 아이콘을 클릭해 설정을 열기
2. "Feature Settings"로 이동
3. **"Enable Checkpoints"** 체크박스를 켜거나 끄기

### 체크포인트를 비활성화할 때

체크포인트는 유용한 안전장치이지만, 다음과 같은 경우 비활성화할 수 있습니다:

- **대규모 저장소**: 매우 큰 코드베이스에서는 추가 저장 공간을 사용할 수 있음
- **성능 고려**: 리소스가 제한된 시스템에서는 약간의 성능 향상이 가능
- **간단한 작업**: 롤백이 필요 없는 간단하고 저위험한 작업


## 변경 사항 보기 및 복원

각 도구 사용 후 다음을 할 수 있습니다:

1. "Compare" 버튼으로 변경된 파일 확인
2. "Restore" 버튼으로 복원 옵션 열기

## 복원 옵션

이전 시점으로 복원하려면:

1. 각 단계 옆의 "Restore" 버튼 클릭
2. 세 가지 옵션 중 선택:
    - **작업과 워크스페이스 복원**: 코드베이스와 작업을 모두 해당 시점으로 되돌림
    - **작업만 복원**: 코드베이스 변경은 유지하고 작업 컨텍스트만 되돌림
    - **워크스페이스만 복원**: 코드베이스를 되돌리되 작업 컨텍스트는 유지

예시: 컴포넌트 스타일을 바꾸는 동안 Cline의 변경이 마음에 들지 않는다면 "워크스페이스만 복원"을 사용해 코드 변경만 되돌리고 대화 컨텍스트는 유지한 채 다른 접근을 시도할 수 있습니다.

## 사용 사례

체크포인트는 Cline을 더 실험적으로 사용할 수 있게 해줍니다. 사람의 코딩은 보통 점진적이지만, AI는 빠르게 큰 변경을 만들 수 있습니다. 체크포인트는 이런 변경을 추적하고 필요 시 되돌릴 수 있게 해줍니다.

### 자동 승인 모드 사용

- 빠른 반복 작업을 위한 안전장치
- 예상치 못한 결과를 손쉽게 되돌릴 수 있음

### 다양한 접근 테스트

- 여러 해결책을 자신 있게 시도
- 서로 다른 구현을 비교
- 정상 동작 상태로 빠르게 복귀
- 다양한 디자인 패턴이나 아키텍처 접근을 탐색할 때 이상적

## 권장 사항

1. 실험할 때 안전장치로 체크포인트 활용
2. 언제든 롤백할 수 있다는 확신으로 자동 승인 모드 더 적극 활용
3. 필요에 따라 선택적으로 복원:
    - 새로 시작하려면 "작업과 워크스페이스 복원"
    - 다른 프롬프트를 시도하되 파일 변경을 유지하려면 "작업만 복원"
    - 대화 컨텍스트를 유지한 채 다른 구현을 시도하려면 "워크스페이스만 복원"

## 메시지 편집과의 관계

[메시지 편집 기능](/features/editing-messages)은 "Restore All" 옵션을 선택할 때 내부적으로 체크포인트를 사용합니다. 즉, 메시지를 수정해 다시 제출할 뿐 아니라 그 시점의 워크스페이스 상태까지 복원할 수 있습니다.

## 체크포인트 삭제

작업 히스토리 메뉴에서 **"Delete All History"** 버튼을 사용하면 모든 체크포인트를 삭제할 수 있습니다. 이 작업은 모든 작업도 함께 삭제합니다. 체크포인트는 VS Code의 globalStorage에 저장됩니다.
