포커스 체인은 Cline의 작업 관리 기능을 강화하는 기능으로, 자동 할 일 목록 생성과 실시간 진행 추적을 제공합니다.

<Frame>
	<img
		src="https://storage.googleapis.com/cline_public_images/docs/assets/2dos.gif"
		alt="실시간 진행 추적이 포함된 포커스 체인 할 일 목록 관리"
	/>
</Frame>

이를 통해 Cline은 장기 작업을 수행하면서 LLM에 전달되는 컨텍스트를 매끄럽게 관리하고, 여러 번의 컨텍스트 윈도우 리셋에도 작업 흐름을 유지할 수 있습니다.

<Tip>
  포커스 체인은 Cline의 [Deep Planning 슬래시 명령](/features/slash-commands/deep-planning)과 특히 잘 어울리며, [계획 수립 과정](/features/plan-and-act)에서 만든 구현 작업의 진행 상황을 매끄럽게 추적합니다.
</Tip>

## 핵심 기능

### 자동 할 일 목록 생성

Cline이 작업을 분석해 다음을 포함한 포괄적인 할 일 목록을 자동으로 생성합니다:
- 마크다운 체크리스트 형식의 명확하고 실행 가능한 항목
- 복잡한 작업을 관리 가능한 단계로 논리적으로 분해
- 작업 진행에 따른 실시간 업데이트

### 사용자 편집 가능 할 일 목록

할 일 목록은 편집 가능한 마크다운 파일로 저장됩니다:
- 선호하는 마크다운 에디터에서 직접 편집
- 사용자가 수정한 내용을 자동 감지
- Cline 워크플로로의 원활한 통합
- 작업 헤더의 편집 버튼으로 빠른 접근

### 시각적 진행 추적

작업 헤더에서 진행 상황을 명확하게 표시합니다:
- **단계 카운터**로 현재 진행률 표시(예: "3/8")
- **완료 항목**은 체크 표시로 명확하게 표시
- **현재 작업**은 강조 표시
- **확장 가능한 뷰**로 전체 할 일 목록 확인

### 스마트 리마인더 시스템

구성 가능한 리마인더로 할 일 목록을 최신 상태로 유지합니다:
- 기본 6개 메시지마다 리마인더(1~100 범위에서 사용자 지정 가능)
- Plan 모드에서 Act 모드로 전환 시 자동 알림
- 할 일 목록을 수동으로 편집했을 때 사용자가 직접 업데이트 요청 가능


## 시작하기

<Steps>
  <Step title="Cline 설정 열기">
      - Cline 사이드바의 톱니바퀴 아이콘 클릭
      - "Features" 섹션으로 이동
  </Step>
  <Step title="포커스 체인 활성화">
      - "Enable Focus Chain" 체크
      - 필요 시 "Remind Cline Interval"(기본: 6 메시지) 조정
  </Step>
  <Step title="새 작업 시작">
      - 새 작업을 시작
      - Cline이 자동으로 할 일 목록을 생성하고 관리
  </Step>
</Steps>

| 설정 | 기본값 | 범위 | 설명 |
|---------|---------|-------|-------------|
| Enable Focus Chain | 비활성화 | On/Off | 향상된 작업 진행 추적 활성화 |
| Remind Cline Interval | 6 | 1-100 메시지 | Cline이 할 일 목록을 업데이트하는 빈도 |

## 사용 예시

#### 1. 작업 시작

포커스 체인을 켠 상태에서 새 작업을 시작하면:

``` markdown User Request
User: "Create a user authentication system for my React app"

Cline: [Analyzes request and creates todo list]
```

#### 2. 할 일 목록 생성

Cline은 작업을 위한 포괄적인 계획을 마크다운 파일로 생성합니다:

```markdown Todo List Created
- [ ] Set up project structure
- [ ] Install authentication dependencies
- [ ] Create user registration component
- [ ] Implement login functionality
- [ ] Add password validation
- [ ] Set up user database schema
- [ ] Write authentication tests
- [ ] Deploy to staging environment
```

#### 3. 진행 추적

Cline이 작업을 진행하는 동안, 작업 헤더에는 실시간 진행 상황이 표시됩니다:

```markdown Todo List Header
[3/8] Implement login functionality     ⌄
```

클릭하여 전체 목록을 확인할 수 있습니다:

```markdown Full Todo List
✓ Set up project structure
✓ Install authentication dependencies
✓ Create user registration component
○ Implement login functionality  ← Currently working
○ Add password validation
○ Set up user database schema
○ Write authentication tests
○ Deploy to staging environment
```

#### 4. 사용자 편집

할 일 목록을 조정해야 하나요? 문제 없습니다.

<Steps>
  <Step title="할 일 목록 열기">
    확장된 todo 뷰에서 편집 버튼을 클릭
  </Step>
  <Step title="마크다운 파일 편집">
    에디터에서 마크다운 파일이 열립니다:

    ```markdown Editing Todo List
    # Focus Chain Todo List for Task abc123

    <!-- Edit this markdown file to update your focus chain todo list -->
    <!-- Use - [ ] for incomplete items and - [x] for completed items -->

    - [x] Set up project structure
    - [x] Install authentication dependencies (e.g., Firebase Auth)
    - [x] Create user registration component
    - [ ] Implement login functionality
    - [ ] Add password reset feature
    - [ ] Set up protected routes
    - [ ] Implement logout functionality
    - [ ] Add user profile page
    - [ ] Write authentication tests
    - [ ] Deploy to staging environment

    <!-- Save this file to update the task's todo list -->
    ```
  </Step>
  <Step title="변경 적용">
    항목을 추가/삭제하거나 순서를 조정
  </Step>
  <Step title="파일 저장">
    Cline이 변경을 자동으로 감지해 반영
  </Step>
</Steps>

## 파일 구조

### 할 일 목록 저장 위치

할 일 목록은 작업 디렉터리의 마크다운 파일로 저장됩니다:

``` markdown
<VSCode Global Storage>/
  tasks/
    <taskId>/
      focus_chain_taskid_<taskId>.md
      ... other task files
```

### 마크다운 포맷

할 일 파일은 표준 마크다운 체크리스트 문법을 사용합니다:

```markdown Example Todo Syntax
# Focus Chain Todo List for Task abc123

<!-- Edit this markdown file to update your focus chain todo list -->
<!-- Use the format: - [ ] for incomplete items and - [x] for completed items -->

- [x] Set up project structure
- [x] Install authentication dependencies
- [ ] Create user registration component
- [ ] Implement login functionality
- [ ] Add password validation
- [ ] Set up user database schema
- [ ] Write authentication tests
- [ ] Deploy to staging environment

<!-- Save this file and the todo list will be updated in the task -->
```


## Plan/Act 모드와의 통합

포커스 체인은 Cline의 [Plan/Act 모드](/features/plan-and-act)와 자연스럽게 연동됩니다:

- **Plan 모드**: 구체적인 단계 제시에 사용할 수 있는 선택적 할 일 목록
- **Act 모드**: Plan 모드에서 전환 시 자동으로 할 일 목록 생성

<Tip>
  복잡한 프로젝트의 경우, Plan 모드에서 접근 방법을 논의하고 다듬은 뒤 Act 모드로 전환해 구현을 시작하세요.
</Tip>

## 모범 사례

<AccordionGroup>
  <Accordion title="효과적인 할 일 목록을 위해">
    1. **명확한 요청으로 시작**
      - 초기 작업 설명을 구체적으로 제공
      - 구체적 요구사항과 제약을 포함
      - 선호하는 기술이나 접근 방식을 언급

    2. **생성된 목록 검토**
      - Cline의 분해가 기대와 맞는지 확인
      - 중요한 단계가 모두 포함됐는지 점검
      - 순서가 프로젝트에 적절한지 확인

    3. **필요 시 편집**
      - 누락된 단계를 추가
      - 불필요한 항목 삭제
      - 더 나은 워크플로를 위해 순서 조정
      - 일반적인 항목에 더 구체적인 세부사항 추가
  </Accordion>
  <Accordion title="복잡한 프로젝트를 위해">
    1. **먼저 Plan 모드 사용**
      - 구현 전에 접근 방식을 논의
      - 대화를 통해 요구사항을 다듬기
      - 작업을 시작할 준비가 되면 Act 모드로 전환

    2. **큰 작업 분해**
      - 복잡한 프로젝트를 더 작고 관리 가능한 작업으로 분해
      - 구성 요소별로 별도의 할 일 목록 생성
      - 한 번에 하나의 핵심 영역에 집중

    3. **정기적 검토**
      - 긴 작업 동안 주기적으로 진행 상황 확인
      - 요구사항 변화에 따라 할 일 목록 업데이트
      - 편집을 통해 변경 사항을 Cline에 전달
  </Accordion>
  <Accordion title="협업을 위해">
    1. **할 일 파일 공유**
      - 할 일 마크다운 파일을 팀원과 공유
      - 프로젝트 문서화를 위해 버전 관리에 포함
      - 프로젝트 계획 논의의 기반으로 활용

    2. **일관된 형식 유지**
      - 표준 마크다운 체크리스트 형식을 따르기
      - 항목 설명을 명확하고 실행 가능하게 유지
      - 할 일 목록 전반에 일관된 용어 사용
  </Accordion>
</AccordionGroup>


## 문제 해결

문제가 있나요? 다음 빠른 해결책을 시도해 보세요:

<AccordionGroup>
  <Accordion title="할 일 목록이 업데이트되지 않나요?">
    - 설정에서 포커스 체인이 활성화되어 있는지 확인
    - 작은 모델이나 낮은 성능의 모델에서는 효과가 떨어질 수 있음
    - 작업 디렉터리의 파일 권한이 올바른지 확인
  </Accordion>
  <Accordion title="할 일 파일을 편집할 수 없나요?">
    - 에디터가 마크다운을 지원하는지 확인
    - VS Code가 디렉터리에 쓰기 권한을 갖는지 확인
  </Accordion>
  <Accordion title="진행 상황이 표시되지 않나요?">
    - 할 일 항목이 올바른 문법(`- [ ]`, `- [x]`)을 사용하는지 확인
    - 마크다운 파일 형식이 올바른지 확인
  </Accordion>
</AccordionGroup>

여전히 해결되지 않나요? Cline에서 [/reportbug](/features/slash-commands/report-bug) 명령으로 도움을 요청하세요.

## 기술 세부사항 (궁금한 분을 위해)

<AccordionGroup>
  <Accordion title="파일 모니터링">
    - 실시간 파일 감시로 할 일 마크다운 파일의 변경을 감지
    - 파일 편집과 UI 업데이트 간 자동 동기화
    - 파일 생성/수정/삭제에 대한 안정적인 처리
  </Accordion>
  <Accordion title="진행 계산">
    - 완료 항목 대비 전체 항목을 동적으로 계산
    - `- [x]`와 `- [X]` 완료 문법 모두 지원
    - 향상된 시각 표시를 위한 유니코드 기호(✓, ○) 사용
  </Accordion>
  <Accordion title="프라이버시 고려사항">
    - 할 일 목록은 VS Code 워크스페이스에 로컬 저장
    - 할 일 내용이 외부 서비스로 전송되지 않음
    - 사용 텔레메트리(설정에서 비활성화 가능)
  </Accordion>
</AccordionGroup>

포커스 체인은 Cline을 개인 프로젝트 매니저로 만들어, 작업을 체계적으로 유지하고 흐름을 잃지 않게 도와줍니다. 다음 프로젝트에서 한번 사용해 보세요!
