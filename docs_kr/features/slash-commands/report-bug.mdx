title: "버그 보고 명령"
sidebarTitle: "/reportbug"

`/reportbug`는 Cline에서 이상한 문제가 발생했을 때 정말 유용한 명령입니다. GitHub 버그 리포트에 필요한 세부사항을 전부 기억할 필요 없이, Cline이 개인 버그 보고 도우미가 되어줍니다.

필요한 정보를 단계적으로 수집한 뒤, 올바른 포맷과 시스템 정보를 포함해 GitHub 이슈 페이지로 바로 생성해 줍니다.

#### `/reportbug` 슬래시 명령 사용하기

뭔가 이상하다고 느껴질 때:

-   채팅에 `/reportbug` 입력
-   Cline이 필요한 정보를 안내합니다:
    -   문제를 요약하는 제목
    -   실제로 발생한 일 vs 기대했던 일
    -   재현 단계
    -   관련 출력이나 오류
    -   문제 해결에 도움이 될 추가 컨텍스트
-   제출 전에 모든 내용을 검토할 수 있습니다
-   승인하면, 시스템 정보가 자동 포함된 완성된 GitHub 이슈가 열립니다

#### 예시

지난주에 큰 파일을 읽을 때 Cline이 계속 타임아웃되는 버그를 겪었습니다. GitHub 템플릿을 다 기억하려고 애쓰는 대신 `/reportbug`를 입력했고, Cline이 전체 과정을 안내해 줬습니다.

무엇을 하려 했는지, 대신 어떤 일이 일어났는지, 정확한 재현 단계까지 질문해 주었습니다. 가장 좋은 점은 OS 버전, Cline 버전 같은 기술 정보가 자동으로 포함된다는 점이었습니다.

몇 초 만에 필요한 정보가 모두 들어간 GitHub 이슈가 생성되어, 따로 정보를 찾아다닐 필요가 없었습니다.
