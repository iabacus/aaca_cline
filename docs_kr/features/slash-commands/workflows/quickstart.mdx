이 튜토리얼에서는 GitHub Pull Request를 리뷰하는 과정을 자동화하는 강력한 워크플로를 만듭니다. 이 예시는 CLI 도구, 파일 분석, 사용자 상호작용을 하나의 매끄러운 과정으로 결합하는 방법을 보여줍니다.

### 사전 준비

*   Cline이 설치되어 있어야 합니다.
*   [GitHub CLI(`gh`)](https://cli.github.com/)가 설치되어 있고 인증되어 있어야 합니다.
*   테스트할 Pull Request가 있는 Git 저장소를 열어야 합니다.

## Pull Request 리뷰 워크플로 만들기

이 워크플로는 PR 세부정보 가져오기, 코드 변경 분석, 리뷰 코멘트 초안 작성 과정을 자동화합니다.

<Steps>
  <Step title="워크플로 파일 생성">
    먼저 프로젝트 전용 워크플로를 위한 디렉터리 구조를 만듭니다.

    1.  프로젝트 루트에 `.clinerules` 폴더를 생성합니다.
    2.  `.clinerules` 안에 `workflows` 폴더를 생성합니다.
    3.  `workflows` 폴더 안에 `pr-review.md` 파일을 생성합니다.
  </Step>

  <Step title="워크플로 내용 작성">
    `pr-review.md` 파일을 열고 다음 내용을 추가하세요. 이 워크플로는 PR 정보를 수집하고 변경을 분석하며 리뷰 제출을 돕습니다.

    ````markdown pr-review.md
    # Pull Request Reviewer

    This workflow helps me review a pull request by analyzing the changes and drafting a review.

    ## 1. Gather PR Information
    First, I need to understand what this PR is about. I'll fetch the title, description, and list of changed files.

    ```bash
    gh pr view PR_NUMBER --json title,body,files
    ```

    ## 2. Examine Modified Files
    Now I will examine the diff to understand the specific code changes.

    ```bash
    gh pr diff PR_NUMBER
    ```

    ## 3. Analyze Changes
    I will analyze the code changes for:
    *   **Bugs:** Logic errors or edge cases.
    *   **Performance:** Inefficient loops or operations.
    *   **Security:** Vulnerabilities or unsafe practices.

    ## 4. Confirm Assessment
    Based on my analysis, I will present my findings and ask how you want to proceed.

    ```xml
    <ask_followup_question>
      <question>I've reviewed PR #PR_NUMBER. Here is my assessment:

    [Insert Analysis Here]

    Do you want me to approve this PR, request changes, or just leave a comment?</question>
      <options>["Approve", "Request Changes", "Comment", "Do nothing"]</options>
    </ask_followup_question>
    ```

    ## 5. Execute Review
    Finally, I will execute the review command based on your decision.

    ```bash
    # If approving:
    gh pr review PR_NUMBER --approve --body "Looks good to me! [Summary of analysis]"

    # If requesting changes:
    gh pr review PR_NUMBER --request-changes --body "Please address the following: [Issues list]"

    # If commenting:
    gh pr review PR_NUMBER --comment --body "[Comments]"
    ```
    ````

    <Note>
      이 워크플로를 실행할 때는 `PR_NUMBER`를 실제 PR 번호로 바꿔야 합니다(예: `/pr-review.md 123`).
    </Note>
  </Step>

  <Step title="워크플로 실행">
    이제 새 워크플로를 실행할 준비가 되었습니다.

    1.  Cline 채팅 패널을 엽니다.
    2.  `/pr-review.md` 뒤에 PR 번호를 입력하고(예: `/pr-review.md 42`) Enter를 누릅니다.
    3.  Cline이 PR 세부정보를 가져오고 코드를 분석한 뒤, 리뷰 제출 전에 결과를 보여줍니다.

    <Tip>
      Cline이 `gh pr view` 같은 명령을 실행할 때 출력이 표시되며 잠시 멈출 수 있습니다. Cline이 내용을 분석하고 워크플로를 계속 진행하도록 **Proceed While Running** 버튼을 눌러야 할 수 있습니다.
    </Tip>
  </Step>
</Steps>

### 다른 일반적인 사용 사례

이 예시는 한 가지일 뿐입니다. 다음과 같은 다양한 작업에 워크플로를 만들 수 있습니다:

*   **컴포넌트 생성:** 새 파일(React 컴포넌트, API 엔드포인트 등)의 보일러플레이트 자동화
*   **테스트 실행:** 테스트 스위트를 실행하고 결과를 요약하는 워크플로
*   **애플리케이션 배포:** `docker`, `kubectl` 같은 도구로 배포 파이프라인 자동화
*   **코드 리팩터링:** 복잡한 리팩터링을 단계별로 안내

Cline의 기능과 여러분의 개발 프로세스를 탐색하며 반복 작업을 효율적인 워크플로로 바꿔보세요.
