자동 승인(Auto Approve)은 Cline이 매번 승인 팝업을 띄우지 않고 수행할 수 있는 작업을 선택하게 해줍니다. 반복적인 안전 작업에서는 방해를 줄이고, 고위험 작업은 여전히 엄격하게 제어할 수 있습니다.

같은 안전 작업에 매번 승인 버튼을 누르고 있다면 자동 승인이 문제를 해결합니다. 중요한 변경은 검토하면서도 불필요한 중단을 줄이는 것이 목표입니다.

## 동작 방식

자동 승인은 **도구 호출 단위**로 평가됩니다. Cline이 파일 읽기/편집, 명령 실행, 브라우저 사용을 하려는 순간 각 카테고리별 자동 승인 설정을 확인합니다.

실제로 중요한 몇 가지 사항:

- **워크스페이스 내부 vs 외부**: “모든 파일 읽기/편집”은 기본 토글이 켜져 있을 때만 동작합니다. 기본 토글이 꺼져 있으면 “모든 파일” 옵션은 효과가 없습니다.

- **터미널 명령**: Cline은 명령을 안전(safe) 또는 승인 필요로 구분합니다. “안전한 명령 실행”은 첫 번째 카테고리만 포함하고, “모든 명령 실행”은 승인 필요 명령까지 확장합니다.

- **알림**: 활성화하면 승인 필요한 경우와 자동 승인된 터미널 명령이 30초 이상 실행될 때 OS 알림을 보냅니다.

<Note>
	[YOLO 모드](/features/yolo-mode)는 이러한 세부 승인 설정을 모두 우회합니다.
</Note>

## 권한 설정

아래 라벨은 자동 승인 메뉴에 표시되는 항목과 동일합니다.

| 설정 | 허용되는 동작 | 비고 |
|--------|-----------------|------|
| 프로젝트 파일 읽기 | 워크스페이스 파일 읽기, 파일 목록, 검색 | 대부분의 작업에 적합한 기본값 |
| 모든 파일 읽기 | 워크스페이스 외부 파일 읽기 | “프로젝트 파일 읽기” 필요 |
| 프로젝트 파일 편집 | 워크스페이스 파일 생성/편집 | 체크포인트 사용 권장 |
| 모든 파일 편집 | 워크스페이스 외부 파일 편집 | “프로젝트 파일 편집” 필요 |
| 안전한 명령 실행 | 안전으로 표시된 명령 실행 | 장시간 실행 가능 |
| 모든 명령 실행 | 승인 필요 명령까지 실행 | “안전한 명령 실행” 필요 |
| 브라우저 사용 | 웹 검색/가져오기용 브라우저 도구 사용 | 프록시 이슈 가능 |
| MCP 서버 사용 | MCP 도구 사용 및 리소스 접근 | 일부 서버는 도구별 자동 승인 있음 |
| 알림 활성화 | 장시간 실행 중인 자동 승인 명령 알림 | 자동 승인 메뉴에서 직접 설정 |

<Warning>
	“모든 파일 읽기/편집”은 기본 토글이 켜져 있을 때만 의미가 있습니다. 워크스페이스 외부 접근을 확장하는 옵션입니다.
</Warning>

## 안전 명령 vs 승인 필요 명령 예시

Cline은 고정된 allowlist로 안전/위험을 구분하지 않습니다. 모델이 명령과 인자를 보고 `requires_approval` 플래그를 지정하고, 자동 승인은 그 플래그를 사용합니다.

아래는 예시일 뿐이며 보장을 의미하지 않습니다.

### 보통 안전한 명령

| 예시 | 보통 안전한 이유 |
|--------|-------------------------|
| `npm run build` | 빌드 출력, 직접 파일 삭제 없음 |
| `npm test` | 테스트 실행 |
| `git status` | 읽기 전용 |
| `ls -la` | 읽기 전용 |
| `cat package.json` | 읽기 전용 |

### 보통 승인 필요

| 예시 | 승인 필요 이유 |
|--------|------------------------------|
| `npm install <pkg>` | 의존성 및 lockfile 변경 |
| `rm -rf <path>` | 파일 삭제 |
| `mv <a> <b>` | 파일 이동(덮어쓰기 가능) |
| `sed -i ...` | 파일 직접 수정 |
| `curl https://...` | 원격 코드 다운로드/실행 |

<Note>
	명령이 안전으로 분류되는지는 정확한 명령, 플래그, 현재 작업에 따라 달라집니다. 확실하지 않다면 명령 자동 승인을 끄고 수동으로 승인하세요.
</Note>

## 알림 활성화

자동 승인된 작업은 길어질 수 있습니다. 알림을 활성화하면 자동 승인된 명령이 오래 실행될 때 알림을 받을 수 있습니다.

**알림 활성화** 토글은 자동 승인 메뉴 하단(구분선 아래)에 있습니다. 승인 권한을 관리하는 위치와 함께 배치되어 쉽게 찾고 조정할 수 있습니다.


## 권장 설정

좋은 기본 설정은 다음과 같습니다:

- **프로젝트 파일 읽기** 활성화
- **프로젝트 파일 편집**, **안전한 명령 실행**, **브라우저 사용**, **MCP 서버 사용**은 필요할 때까지 비활성화

편집을 허용한다면 [체크포인트](/features/checkpoints)로 빠른 롤백을 준비하세요.

민감한 환경(프로덕션 자격 증명, 개인 파일, 회사 기기)에서는 외부 파일 접근과 명령 실행을 엄격히 제한하고 수동 승인으로 진행하세요.
