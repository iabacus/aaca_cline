Cline은 작업 히스토리를 관리하는 도구를 제공하여, 워크스페이스를 효율적으로 정리하고 검색하며 유지할 수 있도록 돕습니다. 시간이 지날수록 작업이 누적되기 때문에 이러한 기능은 생산성에 필수적입니다.

## 작업 히스토리 접근

Cline에서 작업 히스토리를 열고 탐색하는 다양한 방법을 알아보세요. 버튼 클릭, 키보드 단축키, 명령 팔레트 등 편한 방식으로 과거 작업에 접근할 수 있습니다.

작업 히스토리에 접근하는 방법:

1. **Cline 사이드바의 "History" 버튼 클릭**
2. **명령 팔레트 사용**: "Cline: Show Task History" 검색
3. **키보드 단축키**(VS Code 설정에서 구성한 경우)

## 작업 히스토리 인터페이스

작업을 표시하고 관리하는 메인 인터페이스를 살펴봅니다. 레이아웃, 검색 기능, 정렬 옵션, 필터링 도구를 통해 누적된 작업을 효율적으로 탐색할 수 있습니다. 작업 히스토리 뷰는 과거 및 현재 작업을 관리하는 종합 인터페이스입니다.

### 검색 및 필터

히스토리 뷰에는 검색 및 필터 기능이 포함됩니다:

#### 검색 바
- **퍼지 검색**으로 전체 작업 내용 탐색
- 프롬프트, 응답, 코드까지 검색
- 입력과 동시에 결과 필터링
- 결과에서 일치 텍스트 하이라이트

#### 정렬 옵션
작업을 다음 기준으로 정렬할 수 있습니다:
- **최신순**(기본값) - 최근 작업이 먼저 표시
- **오래된순** - 가장 이전 작업부터 표시
- **가장 비싼 작업** - API 비용이 높은 작업 우선
- **토큰 사용량** - 토큰 사용량이 높은 작업 우선
- **가장 관련성 높음** - 검색 시 가장 잘 매칭된 작업

#### 즐겨찾기 필터
- 즐겨찾기 표시된 작업만 보기
- 중요한 작업에 빠르게 접근
- 검색과 결합해 정밀 필터링 가능

## 작업 액션

히스토리의 개별 작업에 대해 수행할 수 있는 다양한 액션을 확인합니다. 다시 열기, 재개, 내보내기, 관리 등 작업을 조작하는 모든 기능을 설명합니다.

히스토리의 각 작업에는 다음과 같은 액션이 있습니다:

### 주요 액션

- **열기**: 작업을 클릭해 Cline 채팅에서 다시 열기
- **재개**: 중단된 작업을 이전 지점부터 계속
- **내보내기**: 대화를 마크다운으로 저장(문서화용)

### 관리 액션

- **즐겨찾기** ⭐: 별 아이콘으로 중요한 작업 표시
- **삭제** 🗑️: 개별 작업 삭제(즐겨찾기는 보호됨)
- **복제**: 기존 작업을 기반으로 새 작업 생성

## ⭐ 작업 즐겨찾기

가장 중요한 작업을 표시하고 보호하는 즐겨찾기 시스템을 익혀보세요. 중요한 작업에 별을 표시해 실수로 삭제되는 것을 방지하고, 참고 구현이나 성공 사례에 빠르게 접근할 수 있습니다.

즐겨찾기 시스템은 중요한 작업을 보존하고 빠르게 찾는 데 도움을 줍니다.

### 즐겨찾기 사용

**즐겨찾기 표시**
- 작업 옆 별 아이콘 클릭
- 즐겨찾기되면 별이 채워짐
- 다시 클릭하면 해제

**보호 기능**
- 즐겨찾기 작업은 실수 삭제에서 보호
- 일괄 삭제 시 기본적으로 즐겨찾기를 건너뜀
- 명시적 확인으로 보호를 해제할 수 있음

**즐겨찾기 활용 사례**
- 보관하고 싶은 참고 구현
- 성공적인 문제 해결 패턴
- 재사용 가능한 코드 스니펫이 있는 작업
- 중요한 프로젝트 마일스톤
- 팀원 학습용 예시

## 작업 메트릭

작업 메트릭을 통해 Cline 사용량을 이해하세요. 토큰 사용량, API 비용 등 다양한 지표를 확인해 워크플로를 최적화하고 리소스를 관리할 수 있습니다.

작업 메트릭을 이해하면 사용 최적화에 도움이 됩니다:

### 사용 가능한 메트릭

- **토큰 사용량**: 소비된 입력/출력 토큰 총량
- **API 비용**: 모델 가격을 기반으로 한 추정 비용
- **체크포인트 수**: 생성된 파일 스냅샷 수

### 메트릭 활용

- **예산 추적**: 작업별 API 비용 모니터링
- **효율성 분석**: 비용이 큰 작업 식별
- **모델 비교**: 모델 간 비용 비교
- **최적화**: 더 효율적으로 만들 수 있는 작업 찾기
