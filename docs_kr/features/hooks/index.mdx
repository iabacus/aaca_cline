훅은 Cline의 워크플로에서 중요한 순간에 사용자 로직을 주입할 수 있게 해줍니다. 실행 전에 작업을 검증하고, 도구 사용을 모니터링하며, Cline의 결정 방식에 영향을 주는 자동 체크포인트라고 생각하면 됩니다.

훅은 개발 중 특정 이벤트가 발생할 때 자동으로 실행됩니다. 각 작업에 대한 상세 정보를 받아 문제를 일으키기 전에 동작을 차단할 수 있고, 이후 AI 결정을 안내할 컨텍스트를 주입할 수도 있습니다.

진짜 힘은 이 기능들을 조합할 때 나옵니다. 예를 들어:

- 문제가 생기기 전에 동작을 중단(예: TypeScript 프로젝트에서 `.js` 파일 생성 차단)
- 발생한 일을 학습해 프로젝트 지식을 축적
- 성능을 모니터링하고 이슈를 조기에 감지
- 분석이나 컴플라이언스를 위해 모든 작업을 추적
- 적절한 시점에 외부 도구나 서비스를 트리거

## 시작하기

<Frame>
	<img src="https://storage.googleapis.com/cline_public_images/hooks.gif" alt="훅 동작 예시" />
</Frame>

<Note>
훅은 Windows, macOS, Linux 전 플랫폼에서 동작합니다. 이 문서의 bash 예시는 모든 플랫폼의 표준 셸(Windows의 Git Bash 또는 WSL 포함)에서 동작합니다.
</Note>

Cline에는 내장 훅 관리 인터페이스가 있어 훅 설정이 쉽습니다. 시작 방법은 다음과 같습니다:

<Steps>
<Step title="훅 인터페이스 접근">
훅 관리 인터페이스로 이동하세요:

<Frame>
  <img src="/assets/hooks/hooks-interface-with-dropdown.png" alt="드롭다운 메뉴가 있는 훅 관리 인터페이스(전역 훅과 프로젝트 훅 표시)" />
</Frame>

1. Cline을 열고(설정에서 훅이 활성화되어 있는지 확인)
2. 상단의 **Hooks** 탭을 찾습니다(옆에 Rules, Workflows)
3. **Hooks**를 클릭해 훅 관리 패널을 엽니다

인터페이스에는 워크스페이스별로 정리된 모든 훅 유형과 기존 훅이 표시됩니다.
</Step>

<Step title="훅 위치 이해">
훅은 인터페이스에서 위치에 따라 자동으로 정리됩니다:

**Global Hooks** - 모든 워크스페이스에 적용:
- `~/Documents/Cline/Hooks/`에 저장
- 개인 코딩 표준과 공통 규칙에 적합

**Project-Specific Hooks** - 현재 프로젝트에만 적용:
- 저장소 내부 `.clinerules/hooks/`에 저장
- 프로젝트별 검증과 팀 워크플로에 적합
- 팀 공유를 위해 버전 관리에 커밋 가능

멀티 루트 워크스페이스는 열린 모든 저장소의 훅을 실행하므로, 하나의 워크스페이스에서 여러 저장소 훅을 쉽게 관리하고 실행할 수 있습니다.
</Step>

<Step title="첫 번째 훅 만들기">
직관적인 인터페이스로 훅을 만들 수 있습니다:

<Frame>
  <img src="/assets/hooks/hooks-empty-state.png" alt="훅이 없을 때의 인터페이스: 전역 훅과 프로젝트 훅의 New hook... 드롭다운 표시" />
</Frame>

1. **위치 선택**: Global Hooks 또는 프로젝트 훅 중 선택
2. **훅 유형 선택**: 선택한 위치에서 **"New hook..."** 드롭다운 클릭
3. **훅 유형 고르기**: 해당 위치에서 아직 생성되지 않은 훅 유형만 표시됩니다. 훅 디렉터리당 훅 유형은 하나만 허용되므로, 드롭다운이 남은 유형만 보여줍니다.

<Frame>
  <img src="/assets/hooks/new-hook-dropdown.png" alt="UserPromptSubmit이 선택된 드롭다운으로 새 훅 생성" />
</Frame>

4. **훅 검토 및 편집**: 연필 아이콘을 클릭해 훅 코드를 검토하고 사용자 로직 추가
5. **훅 활성화**: 동작을 이해하고 승인했다면 토글 스위치를 켭니다

<Frame>
  <img src="/assets/hooks/hook-controls.png" alt="각 훅의 토글, 편집, 삭제 버튼이 보이는 훅 관리 컨트롤" />
</Frame>

<Warning>
훅을 활성화하기 전에 항상 코드를 검토하세요. 훅은 워크플로 중 자동 실행되므로, 활성화 전에 동작을 이해하는 것이 중요합니다.
</Warning>
</Step>

<Step title="훅 테스트">
훅을 개발하고 다듬으려면 테스트 중 여러 번 트리거해야 합니다. 훅 유형마다 다른 이벤트에 의해 트리거됩니다. 예:

- **TaskStart** 훅: 새 작업 시작 시
- **PreToolUse** 훅: Cline이 파일 편집 같은 도구를 실행하기 전
- **PostToolUse** 훅: 도구 실행 완료 후
- **UserPromptSubmit** 훅: 메시지를 제출할 때

각 훅 유형이 언제 트리거되는지와 효과적으로 테스트하는 방법은 [훅 레퍼런스](/features/hooks/hook-reference) 문서를 참고하세요. 훅별 트리거 조건과 개발 중 호출 방법의 예시가 포함되어 있습니다.
</Step>
</Steps>

<Tip>
처음에는 정보만 로깅하는 간단한 훅으로 시작한 뒤, 복잡한 검증 로직을 구축하세요. 데이터 구조와 타이밍을 이해하는 데 도움이 됩니다.
</Tip>

## CLI 지원

훅은 [Cline CLI](/cline-cli/overview)에서도 사용할 수 있습니다. 명령줄에서 작업을 실행할 때 훅을 활성/비활성화할 수 있습니다:

```bash
# 작업에 훅 활성화
cline "What does this repo do?" -s hooks_enabled=true

# CLI로 전역 훅 설정
cline config set hooks-enabled=true
cline config get hooks-enabled
```

이를 통해 훅을 자동화 워크플로, CI/CD 파이프라인, 헤드리스 작업 실행에 통합할 수 있습니다.

<Note>
CLI의 훅은 macOS와 Linux에서만 지원됩니다. Windows는 아직 지원되지 않습니다.
</Note>

## 관련 기능

훅은 다른 Cline 기능을 보완합니다:

- [Cline Rules](/features/cline-rules): 훅이 강제할 수 있는 상위 지침 정의
- [체크포인트](/features/checkpoints): 훅이 이슈를 놓쳤을 때 변경을 롤백
- [자동 승인](/features/auto-approve): 자동 작업에 대한 안전망으로 훅과 잘 어울림
- [Cline CLI](/cline-cli/overview): 터미널 기반/자동화 워크플로에서 훅 사용
