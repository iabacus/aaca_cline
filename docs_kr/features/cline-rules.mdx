Cline Rules는 시스템 수준의 지침을 Cline에 제공하는 기능입니다. 프로젝트별 또는 모든 대화에 공통으로 적용할 컨텍스트와 선호 사항을 지속적으로 포함하는 방식으로 이해하면 됩니다.

## 규칙 만들기

Rules 탭의 `+` 버튼을 클릭하면 새 규칙을 만들 수 있습니다. IDE에서 새 파일이 열리며, 여기에 규칙을 작성하면 됩니다.

<Frame>
	<img src="https://storage.googleapis.com/cline_public_images/docs/assets/cline-rules.png" alt="규칙 만들기" />
</Frame>

파일을 저장하면:

-   워크스페이스 규칙인 경우 프로젝트의 `.clinerules/` 디렉터리에 저장됩니다
-   전역 규칙인 경우 전역 규칙 디렉터리에 저장됩니다:

### 전역 규칙 디렉터리 위치

전역 규칙 디렉터리 위치는 운영체제에 따라 다릅니다:

| 운영체제 | 기본 위치 | 비고 |
|------------------|------------------|-------|
| **Windows** | `Documents\Cline\Rules` | 시스템 문서 폴더 사용 |
| **macOS** | `~/Documents/Cline/Rules` | 사용자 문서 폴더 사용 |
| **Linux/WSL** | `~/Documents/Cline/Rules` | 일부 시스템에서는 `~/Cline/Rules`로 대체될 수 있음 |

> **Linux/WSL 사용자 참고**: `~/Documents/Cline/Rules`에서 전역 규칙을 찾을 수 없다면 `~/Cline/Rules`를 확인하세요. 시스템 설정이나 Documents 디렉터리 존재 여부에 따라 위치가 달라질 수 있습니다.

채팅에서 [`/newrule` 슬래시 명령](/features/slash-commands/new-rule)을 사용해 Cline이 규칙 파일을 만들어 주도록 할 수도 있습니다.

```markdown Example Cline Rule Structure [expandable]
# 프로젝트 가이드라인

## 문서화 요구사항

-   기능 변경 시 /docs 관련 문서를 업데이트
-   README.md를 새로운 기능과 동기화
-   CHANGELOG.md에 변경 로그 항목 유지

## 아키텍처 결정 기록(ADR)

다음과 같은 경우 /docs/adr에 ADR을 생성:

-   주요 의존성 변경
-   아키텍처 패턴 변경
-   신규 통합 패턴
-   데이터베이스 스키마 변경
    /docs/adr/template.md 템플릿을 따르세요

## 코드 스타일 및 패턴

-   OpenAPI Generator로 API 클라이언트 생성
-   TypeScript axios 템플릿 사용
-   생성된 코드는 /src/generated에 배치
-   상속보다 합성 선호
-   데이터 접근에 리포지토리 패턴 사용
-   /src/utils/errors.ts의 에러 처리 패턴 준수

## 테스트 기준

-   비즈니스 로직에 단위 테스트 필수
-   API 엔드포인트에 통합 테스트
-   핵심 사용자 흐름에 E2E 테스트
```

### 주요 이점

1. **버전 관리 가능**: `.clinerules` 파일이 프로젝트 소스 코드의 일부가 됩니다
2. **팀 일관성**: 모든 팀 구성원에게 일관된 동작을 보장합니다
3. **프로젝트 맞춤형**: 프로젝트별 요구에 맞춘 규칙과 표준을 적용합니다
4. **조직 지식 보존**: 프로젝트 표준과 관행을 코드로 유지합니다

`.clinerules` 파일은 프로젝트 루트 디렉터리에 두세요:

```
your-project/
├── .clinerules
├── src/
├── docs/
└── ...
```

반면 Cline의 시스템 프롬프트는 사용자가 편집할 수 없습니다([여기에서 확인](https://github.com/cline/cline/blob/main/src/core/prompts/system.ts)). 프롬프트 엔지니어링 모범 사례에 대해 더 알아보려면 [이 리소스](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)를 참고하세요.

### AGENTS.md 표준 지원

Cline은 워크스페이스 루트의 `AGENTS.md` 파일을 자동으로 감지하여 [AGENTS.md](https://agents.md/) 표준도 지원합니다(추가적으로 Cline Rules의 대체 수단으로). 이를 통해 다양한 AI 코딩 도구에서 동일한 규칙 파일을 사용할 수 있습니다.

```
your-project/
├── AGENTS.md
├── src/
└── ...
```

### 효과적인 Cline Rules 작성 팁

-   명확하고 간결하게: 쉬운 언어로 모호함을 줄이세요.
-   원하는 결과에 집중: 구체적인 단계보다 목표 결과를 설명하세요.
-   테스트와 반복: 워크플로에 맞는 최적의 방식을 실험하세요.

### .clinerules/ 폴더 시스템

```
your-project/
├── .clinerules/              # 활성 규칙이 들어 있는 폴더
│   ├── 01-coding.md          # 핵심 코딩 표준
│   ├── 02-documentation.md   # 문서화 요구사항
│   └── current-sprint.md     # 현재 작업에 특화된 규칙
├── src/
└── ...
```

Cline은 `.clinerules/` 디렉터리 안의 **모든 Markdown 파일**을 자동으로 처리해 하나의 규칙 세트로 결합합니다. (선택 사항인) 숫자 접두사는 파일을 논리적 순서로 정리하는 데 도움이 됩니다.

#### 규칙 뱅크 사용

여러 컨텍스트나 팀이 있는 프로젝트에서는 규칙 뱅크 디렉터리를 유지할 수 있습니다:

```
your-project/
├── .clinerules/              # 활성 규칙 - 자동 적용
│   ├── 01-coding.md
│   └── client-a.md
│
├── clinerules-bank/          # 사용 가능하지만 비활성인 규칙 저장소
│   ├── clients/              # 클라이언트별 규칙 세트
│   │   ├── client-a.md
│   │   └── client-b.md
│   ├── frameworks/           # 프레임워크별 규칙
│   │   ├── react.md
│   │   └── vue.md
│   └── project-types/        # 프로젝트 유형 표준
│       ├── api-service.md
│       └── frontend-app.md
└── ...
```

#### 폴더 접근 방식의 장점

1. **컨텍스트 기반 활성화**: 필요한 규칙만 뱅크에서 활성 폴더로 복사
2. **유지보수 용이**: 다른 규칙에 영향을 주지 않고 개별 파일만 업데이트
3. **팀 유연성**: 팀원별로 현재 작업에 맞는 규칙을 활성화 가능
4. **잡음 감소**: 활성 규칙 세트를 집중적이고 관련성 있게 유지

#### 사용 예시

클라이언트 프로젝트 전환:

```bash
# Client B 프로젝트로 전환
rm .clinerules/client-a.md
cp clinerules-bank/clients/client-b.md .clinerules/
```

기술 스택 변경:

```bash
# 프론트엔드 React 프로젝트
cp clinerules-bank/frameworks/react.md .clinerules/
```

#### 구현 팁

-   개별 규칙 파일을 특정 관심사에 집중시키세요
-   규칙 목적이 명확한 설명적 파일명을 사용하세요
-   활성 `.clinerules/` 폴더는 git ignore 처리하고 `clinerules-bank/`만 추적하는 방식을 고려하세요
-   자주 쓰는 규칙 조합을 빠르게 활성화하는 팀 스크립트를 만들어 두세요

폴더 시스템은 Cline 규칙을 정적 문서에서 벗어나, 팀의 변화하는 컨텍스트와 요구에 맞춰 적응하는 동적 지식 시스템으로 바꿔줍니다.

### 토글 가능한 팝오버로 규칙 관리

단일 `.clinerules` 파일과 폴더 시스템을 더 쉽게 관리할 수 있도록, Cline v3.13에서는 채팅 인터페이스에서 바로 접근 가능한 전용 팝오버 UI를 제공합니다.

채팅 입력 필드 아래에 위치한 이 팝오버로 다음을 할 수 있습니다:

-   **활성 규칙 즉시 확인**: 사용자 설정의 전역 규칙과 워크스페이스 규칙(`.clinerules` 파일 또는 폴더 내용) 활성 상태를 확인
-   **규칙 빠른 토글**: 워크스페이스 `.clinerules/` 폴더 내 특정 규칙 파일을 한 번의 클릭으로 켜고 끌 수 있음. 필요한 경우에만 `react-rules.md`나 `memory-bank.md` 같은 컨텍스트 전용 규칙을 활성화하는 데 유용합니다.
-   **규칙 추가/관리 간편화**: 워크스페이스에 `.clinerules` 파일이나 폴더가 없으면 빠르게 생성하거나, 기존 폴더에 새 규칙 파일을 추가 가능

이 UI는 대화 중에 파일이나 설정을 직접 수정하지 않고도 컨텍스트를 전환하고 다양한 지침 세트를 관리할 수 있게 해줍니다.
