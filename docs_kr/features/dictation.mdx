받아쓰기는 AI와의 작업 방식을 바꿉니다. 복잡한 생각을 타이핑하는 대신, 자연스럽게 말로 전달해 전체 의도를 공유할 수 있습니다. 단순히 속도가 빠른 것뿐 아니라(물론 음성이 더 빠릅니다), 타이핑으로는 불가능한 유연한 협업을 가능하게 합니다.

## 음성이 모든 것을 바꾸는 이유

타이핑을 할 때 우리는 스스로를 편집합니다. 복잡한 아이디어를 단순화하고, 맥락을 생략하며, 미묘한 뉘앙스를 잃습니다. 말로 할 때는 머릿속의 모든 것을 공유하게 됩니다. 전체 문제, 제약, 걱정되는 엣지 케이스까지요.

[Plan 모드](/features/plan-and-act)에서는 받아쓰기를 자주 사용해 빠른 왕복 논의를 하세요. 신중하게 구조화된 프롬프트를 타이핑하기 대신 문제를 생각하고 말하면 됩니다. Cline이 추가 질문을 하면 바로 답하고, 견고한 계획이 나올 때까지 반복하세요.

타이핑의 마찰이 진정한 협업을 가로막고 있었습니다. 음성은 그 마찰을 없앱니다.

## 시작하기

**받아쓰기 활성화:**
1. 설정 → Features → Dictation 으로 이동
2. "Enable Dictation" 토글을 켭니다
3. 안내에 따라 Cline 계정에 로그인합니다
4. FFmpeg를 아직 설치하지 않았다면 설치합니다(Cline이 안내합니다)

활성화되면 채팅 입력 영역에 마이크 버튼이 표시됩니다.

**받아쓰기 사용 방법:**
- 마이크 버튼을 클릭해 녹음을 시작
- 자연스럽게 말하기
- 다시 클릭해 녹음 종료
- 채팅에 전사 결과가 나타날 때까지 기다리기

<Tip>
받아쓰기는 설정된 어떤 AI 모델과도 동작합니다. 전사는 Cline 서비스에서 처리되지만, 대화는 현재 사용 중인 모델로 계속됩니다.
</Tip>

## 시스템 요구사항

<Note>
받아쓰기는 현재 Windows에서 제공되지 않습니다. 향후 릴리스에서 지원될 예정입니다.
</Note>

받아쓰기는 모든 플랫폼에서 음성을 캡처하기 위해 FFmpeg를 사용합니다:

- **macOS**: FFmpeg (Homebrew로 설치: `brew install ffmpeg`)
- **Linux**: FFmpeg (apt로 설치: `sudo apt-get install ffmpeg`)

FFmpeg가 설치되어 있지 않으면 Cline이 자동으로 감지하고 한 번의 클릭으로 설치를 안내합니다.

## 받아쓰기가 빛나는 순간

### Plan 모드 대화

받아쓰기는 [Plan 모드](/features/plan-and-act) 대화에 가장 적합합니다. 신중하게 프롬프트를 구성하는 대신 다음을 할 수 있습니다:

- 전체 문제 컨텍스트를 한 번에 말로 전달
- Cline의 질문에 즉시 응답
- 타이핑 마찰 없이 아이디어 반복
- 생각을 말로 풀어내며 Cline이 듣게 하기

2~3분 동안 연속으로 말하면서, 만들고 싶은 것의 전체 맥락, 제약 조건, 직면한 구체적 문제를 설명하며 계획 세션을 시작해 보세요.

### 복잡한 문제 설명

어떤 문제는 타이핑으로 설명하기 어렵습니다. 예를 들어:
- 엣지 케이스가 많은 다단계 워크플로
- 여러 시스템에 걸친 통합 이슈
- 특정 재현 단계가 필요한 성능 문제
- 세밀한 맥락이 필요한 UI/UX 문제

말로 설명하면 "아, 그리고 이것도..." 같은 중요한 디테일까지 자연스럽게 전달할 수 있습니다.

### 코드 리뷰 및 디버깅

코드를 리뷰하거나 버그를 설명할 때, 음성은 사고 과정을 그대로 전달할 수 있게 합니다:
- "이 함수는 괜찮아 보이는데, 여기서 문제가 생길 것 같아요..."
- "이 부분일 수도 있고, 저쪽 영역일 수도 있고요..."
- "X와 Y를 해봤는데 둘 다 안 된 이유가..."

최종 질문만 던지는 것이 아니라 전체 디버깅 여정을 공유할 수 있습니다.

## 기술 요구사항

**시스템 요구사항:**
- FFmpeg가 시스템에 설치되어 있어야 함
- 활성 인터넷 연결
- 전사 크레딧이 있는 Cline 계정

**오디오 품질:**
- Opus 코덱의 WebM 형식으로 녹음
- 16kHz 샘플 레이트, 모노 오디오
- 음성 인식에 최적화됨

**프라이버시:**
- 오디오는 로컬에서 녹음
- 전사에는 오디오 파일만 전송
- 전사 후 오디오 저장 없음
- 임시 파일은 자동으로 정리됨

## 비용 및 크레딧

음성 전사는 Cline 계정에서 분당 $0.006 비용이 듭니다. 대부분의 사용자에게는 세션당 몇 센트 수준입니다.

5분짜리 계획 대화는 약 3센트 정도입니다. 음성을 많이 쓰는 사용자도 보통 월 몇 달러를 넘기지 않습니다.

<Note>
가격은 실험 중이며, 서비스 개선에 따라 변경될 수 있습니다.
</Note>

## 모범 사례

**자연스럽게 말하기**
타이핑하듯 말하려 하지 마세요. 평소 대화 톤으로, 완벽한 문법은 신경 쓰지 않아도 됩니다.

**먼저 컨텍스트 제공**
큰 그림을 먼저 이야기한 뒤 세부로 들어가세요. 예: "실시간 데이터를 처리해야 하는 React 앱을 만들고 있는데, WebSocket 연결에서 성능 문제가 있어요..."

**탐색용으로 음성 사용**
무엇이 필요한지 확실하지 않을 때 받아쓰기가 유용합니다. 문제를 말로 풀어내며 대화를 발전시키세요.

**텍스트와 조합**
모든 걸 음성으로 할 필요는 없습니다. 복잡한 설명과 컨텍스트는 음성으로, 빠른 후속 질문이나 코드 스니펫은 텍스트로 처리하세요.

## 문제 해결

**마이크가 동작하지 않음**
- IDE의 마이크 접근 권한을 확인
- FFmpeg가 제대로 설치되어 있는지 확인
- VS Code/에디터를 새로고침

**전사 품질 저하**
- 명확하고 일반적인 볼륨으로 말하기
- 가능하면 배경 소음 줄이기
- 마이크 설정 확인

**연결 문제**
- 인터넷 연결 확인
- 방화벽이 Cline 서버를 차단하는지 확인
- Cline 계정에서 로그아웃 후 다시 로그인

**인증 문제**
- 인증 오류가 보이면 Cline 계정에서 로그아웃 후 재로그인
- 계정에 전사 크레딧이 충분한지 확인
- 인터넷 연결이 안정적인지 확인

**오디오 녹음 문제**
- FFmpeg가 제대로 설치되어 접근 가능한지 확인
- 브라우저/IDE의 마이크 권한 확인
- 오디오 캡처가 실패하면 에디터 재시작

## AI 협업의 미래

생각을 말하는 속도로 전달할 수 있으면, 자기 검열이 줄어듭니다. 전체 컨텍스트, 엣지 케이스, "만약" 시나리오까지 공유하게 됩니다. 이는 더 나은 해결책과 더 적은 추가 질문을 의미합니다.
