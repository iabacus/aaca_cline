Cline은 작업에 제출된 채팅 메시지를 이후에 편집할 수 있게 해줍니다. 새 작업을 시작하지 않고도 요청을 다듬을 수 있어, 워크플로를 크게 방해하지 않으면서 더 나은 결과를 얻을 수 있습니다.

## 언제 메시지를 편집하나요?

다음과 같은 경우 메시지 편집이 유용합니다:

-   원하는 결과가 나오지 않았을 때
-   요청을 더 잘 표현할 방법이 떠올랐을 때
-   추가 정보나 컨텍스트를 보강해야 할 때
-   원래 메시지에 오타나 오류가 있을 때

## 메시지 편집 방법

1. 대화 내에서 메시지(초기 작업 메시지 제외)를 클릭
2. 필요한 대로 텍스트를 수정
3. 복원 옵션을 사용해 요청을 다시 제출

## 복원 옵션

메시지를 편집하면 두 가지 복원 옵션을 사용할 수 있습니다:

### 채팅 복원

"채팅 복원" 옵션은:

-   작업 상태만 복원
-   편집한 메시지로 API 요청을 다시 제출
-   그 시점까지의 파일 변경 사항을 유지
-   워크스페이스 현재 상태를 유지하고 싶을 때 유용

### 모두 복원

"모두 복원" 옵션은:

-   작업 상태와 워크스페이스 상태를 모두 복원
-   편집한 메시지로 API 요청을 다시 제출
-   대화 시점의 워크스페이스 상태로 되돌림
-   내부적으로 [체크포인트](/features/checkpoints)를 사용해 워크스페이스를 복원
-   완전히 다른 접근을 시도할 때 유용

## 키보드 단축키

메시지 편집 시 다음 단축키를 사용할 수 있습니다:

-   **Escape**: 변경 없이 편집 모드 종료
-   **Enter**: 작업만 복원(= "채팅 복원")
-   **Cmd/Ctrl + Enter**: 작업과 워크스페이스 복원(= "모두 복원")
-   **Shift + Enter**: 메시지에 줄바꿈 추가

## 모범 사례

-   요청의 작은 수정에는 메시지 편집을 사용
-   방향 전환이 큰 경우에는 새 작업을 시작하는 것을 고려
-   "모두 복원"을 사용할 때, 해당 메시지 이후의 파일 변경이 되돌려짐을 유의
-   대화 초반 메시지를 편집하면 큰 진행 손실을 줄일 수 있음
